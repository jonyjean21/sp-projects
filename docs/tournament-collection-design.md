## 自動収集方法の比較

### X（Twitter）API 直接利用

| プラン | 費用 | 読み取り上限 | 検索範囲 | 判定 |
|--------|------|-------------|---------|------|
| Free | 無料 | 約50件/月 | なし | **使い物にならない** |
| Basic | $200/月 | 10K〜15K件/月 | 直近7日 | **高すぎる** |
| Pro | $5,000/月 | 約100万件/月 | 全期間 | 論外 |
| Pay-Per-Use | 従量制 | 可変 | 可変 | クローズドβ（2025/12〜） |

**結論: X API直接利用はコスパが悪すぎるため不採用**

### IFTTT（Pro $2.99/月）

| 項目 | 内容 |
|------|------|
| 費用 | **$2.99/月**（Pro、約450円）、$8.99/月（Pro+） |
| X API別途契約 | **不要**（IFTTTが自社でAPI契約を維持） |
| X連携 | 利用可能（有料ユーザーのみ） |
| トリガー | 新しいメンション、キーワード検索、特定ユーザーの投稿 |
| ポーリング | 5分ごと（Pro） |
| 制約 | 1回のチェックで最大10ツイート / 保護ツイートは対象外 |
| アクション | Google スプレッドシートに行を追加 |

**メリット:** 安い、設定が簡単、X検索→スプシ連携がノーコードで可能
**デメリット:** 1チェック10件の制限、検索精度に限界

### Zapier

| 項目 | 内容 |
|------|------|
| 費用 | 無料プラン（100タスク/月）〜有料 |
| X連携 | 2026年2月に復活。ただし自前のX Developer Appが必要 |
| トリガー | New Mention, New Post from Query, New Post from User |
| アクション | Google Sheets行追加、WordPress投稿等 |

**メリット:** X連携が復活（2026年2月16日）、8,000アプリと連携可能
**デメリット:** X Developer App取得が必要 = **X API Basic $200/月が別途必要**。IFTTTに比べて圧倒的に割高
**結論: 不採用**

### Google Alerts（無料）

| 項目 | 内容 |
|------|------|
| 費用 | **無料** |
| 対象 | Google検索にインデックスされたウェブページ |
| 配信先 | メール or RSS |
| スプシ連携 | IMPORTFEED関数で直接取り込み可能 |
| 設定キーワード例 | 「モルック 大会」「モルック 大会 参加者募集」 |

**メリット:** 無料、設定5分、ウェブ上の大会告知記事を自動キャッチ
**デメリット:** X上のポストは拾えない、リアルタイム性は低い

### RSS収集（無料）

| 項目 | 内容 |
|------|------|
| 対象 | molkky.jp、jajapatatas.com のRSS |
| スプシ連携 | IMPORTFEED関数、またはGASで定期取得 |

**メリット:** 公式情報を自動取得できる
**デメリット:** RSS提供していないサイトは対象外

## おすすめ構成: 3層収集パイプライン

費用・手間・PC時間の少なさを考慮した最適解。

```
┌─────────────────────────────────────────────┐
│            大会情報 収集パイプライン            │
├─────────────────────────────────────────────┤
│                                             │
│  Layer 1: Google Alerts（無料・自動）         │
│  ├─ 「モルック 大会」等のキーワード設定        │
│  ├─ RSS配信 → Google スプレッドシートに蓄積    │
│  └─ ウェブ上の告知記事を自動キャッチ          │
│                                             │
│  Layer 2: IFTTT Pro（$2.99/月・自動）         │
│  ├─ X検索トリガー: #モルック + 大会キーワード  │
│  ├─ 特定アカウント監視: @Molkky_Japan 等      │
│  └─ → Google スプレッドシートに自動追記       │
│                                             │
│  Layer 3: 手動チェック（週1回・補完）          │
│  ├─ molkky.jp/tournament/ を目視確認         │
│  ├─ jajapatatas.com を目視確認              │
│  └─ XのTL巡回で拾った情報をスプシに追記      │
│                                             │
├─────────────────────────────────────────────┤
│  ▼ すべて同じ Google スプレッドシートに集約     │
│  ▼ 既存の GAS → WordPress 自動投稿で公開     │
└─────────────────────────────────────────────┘
```

## 段階的導入プラン

### Phase 1（今すぐ・無料・15分）— Google Alerts 設定

1. https://www.google.co.jp/alerts にアクセス
2. 以下のキーワードでアラートを作成:
   - 「モルック 大会」
   - 「モルック 参加者募集」
   - 「モルック 大会 開催」
3. 配信先を「RSSフィード」に設定
4. RSSのURLをスプレッドシートで `=IMPORTFEED("URL")` で取り込み

**これだけでウェブ上の新着情報が自動的にスプシに入る。**

### Phase 2（翌週・$2.99/月）— IFTTT Pro でX監視

1. IFTTT Pro に登録（$2.99/月、約450円）
2. Appletを作成（Pro は最大20個まで）:
   - Applet 1: `#モルック` 検索 → Google Sheets
   - Applet 2: `@Molkky_Japan` のツイート → Google Sheets
   - Applet 3: `@molkkycalendar` のツイート → Google Sheets
   - Applet 4: `モルック 大会` 検索 → Google Sheets
   - Applet 5: 全国モルックカレンダーのRSS → Google Sheets
   - Applet 6: Google Alerts RSS → Google Sheets

**X上の大会情報が5分ごとに自動でスプシに流れ込む。**

> 参考Applet: https://ifttt.com/applets/hHiMAk5C （ハッシュタグ→スプシ）

### Phase 3（余裕があれば）— RSS + GAS で公式サイト監視

1. molkky.jp、jajapatatas.com のRSSフィードURLを確認
2. GASで定期的にRSSを取得 → スプレッドシートに追記
3. 重複排除ロジックを追加

## 運用上の注意

- **Google Sheetsの2,000行制限**: IFTTTはシートが2,000行に達すると新規シートを自動作成。GASとの連携でシート名を考慮する
- **重複排除**: 複数Appletが同じツイートを拾う可能性あり → GAS側でURL等で重複チェック
- **ノイズ除去**: 「モルック 大会」は関係ない投稿も入る → GASでフィルタリングロジックを追加
