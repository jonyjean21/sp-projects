# 記事投稿の半自動化 設計メモ

> 作成日: 2026-02-24

## 現状

- 記事の半自動化は未着手
- 寄稿含め週2〜3回更新しているが、すべて手動
- PC時間が限られている → 1記事あたりの工数削減が直接的に効く
- 大会情報のGAS→WordPress自動投稿は稼働済み → 同じ基盤を流用可能

## 記事タイプの整理

記事を「定型」「半定型」「非定型」に分け、それぞれに合った効率化を行う。

| タイプ | 例 | 自動化の余地 |
|--------|---|-------------|
| **定型** | チーム紹介、YouTubeチャンネル紹介 | 高（テンプレート+データ入力で生成可能） |
| **半定型** | 大会レポート、寄稿記事 | 中（フォーマットは決まっているが内容はユニーク） |
| **非定型** | コラム、解説記事、ニュース | 低（AI下書き→人間レビューが現実的） |

## 自動化フロー設計

### 定型記事（チーム紹介等）

```
Googleフォーム（チーム名、活動地域、メンバー数、紹介文、画像）
  │
  ▼ GAS
テンプレートに流し込み → WordPress下書き自動作成
  │
  ▼
孔雀がスマホで確認 → 公開
```

**ポイント:**
- SWELLのブロック構成に合わせたHTMLテンプレートをGAS内に持つ
- フォームの項目 = 記事の構成要素なので、テンプレートに流し込むだけで記事が完成
- 画像もWP REST APIでメディアアップロード → 記事に自動埋め込み

### 半定型記事（大会レポート・寄稿）

```
寄稿フォーム or スプレッドシート入力
  │
  ▼ GAS
WordPress下書き作成（基本フォーマット適用）
  │
  ▼
孔雀が微調整 → 公開
```

**ポイント:**
- 寄稿フロー（contribution-flow-design.md）のPhase 2と同じ仕組み
- 大会レポートは大会情報スプレッドシートと連携可能（大会名・日時・場所は既存データを引用）

### 非定型記事（コラム・解説）

```
テーマ・キーワード・メモ（スマホからメモアプリ or フォーム）
  │
  ▼ Claude等のAI
下書き生成
  │
  ▼
孔雀がレビュー・編集 → WordPress投稿 → 公開
```

**ポイント:**
- 完全自動化ではなく「AI下書き→人間レビュー」の半自動
- Claudeのプロンプトテンプレートを用意しておく（トーン、文字数、構成の指定）
- 品質担保のため必ず人間のレビューを挟む

## 共通基盤: WordPress REST API + GAS

3つのフローすべてに共通する技術基盤:

```
データソース（フォーム / スプシ / AI生成テキスト）
  │
  ▼
GAS（テンプレート適用 + HTML生成）
  │
  ▼
WordPress REST API
  ├── POST /wp-json/wp/v2/posts    → 下書き投稿作成
  ├── POST /wp-json/wp/v2/media    → 画像アップロード
  └── 認証: アプリケーションパスワード
  │
  ▼
LINE Notify → 孔雀のスマホに通知
```

大会情報の自動投稿で使っているGASの仕組みを拡張すればよい。

## SWELLテンプレート設計

SWELL固有のブロック（ふきだし、ボックス、ステップ等）を活用するため、GASで生成するHTMLにはSWELLのカスタムクラスを含める。

例: チーム紹介テンプレート
```html
<!-- wp:heading -->
<h2>チーム名</h2>
<!-- /wp:heading -->

<!-- wp:table {"className":"is-style-stripes"} -->
<figure class="wp-block-table is-style-stripes"><table>
<tbody>
<tr><td>活動地域</td><td>{region}</td></tr>
<tr><td>メンバー数</td><td>{members}名</td></tr>
<tr><td>活動頻度</td><td>{frequency}</td></tr>
</tbody>
</table></figure>
<!-- /wp:table -->

<!-- wp:paragraph -->
<p>{description}</p>
<!-- /wp:paragraph -->
```

## 導入の優先順

| 優先 | やること | 工数目安 | 効果 |
|------|---------|---------|------|
| 1 | 寄稿フォーム統一（Phase 1） | 30分 | 連絡手段統一・管理一元化 |
| 2 | 寄稿のGAS→WP連携（Phase 2） | 2〜4時間 | 手動転記ゼロ |
| 3 | 定型記事テンプレート（チーム紹介） | 2〜3時間 | テンプレ記事の量産体制 |
| 4 | AI下書き生成のプロンプト整備 | 1〜2時間 | 非定型記事の執筆時間短縮 |
