- 各投稿・固定ページごとにmeta description設定可能

- 構造化データ（JSON-LD）の自動出力

## PC時間が少ない人向け 優先施策 TOP3

### 1. Search Console で「お宝キーワード」を発掘（30分）

**やること:**
1. Search Console → 検索パフォーマンス → クエリタブ
2. 「表示回数が多いのにクリック率(CTR)が低い」クエリを探す
3. そのクエリに対応する記事のタイトル・meta descriptionを改善

**なぜ効くか:**
- すでにGoogleに表示されているが、クリックされていない = タイトル改善だけで流入増
- 新記事を書くより圧倒的に工数が少ない

**具体例:**
- 「モルック ルール」で表示回数100、CTR 3% → タイトルを「【2025年最新】モルックのルールを完全解説」に変更
- → CTR 3% → 8% になれば、それだけで流入が2.7倍

### 2. 主要ページのmeta description を最適化（1時間）

**やること:**
1. 表示回数TOP10のページを確認
2. 各ページのmeta descriptionが適切か確認（SEO SIMPLE PACK の設定欄）
3. 検索意図に合った、クリックしたくなるdescriptionに書き換え

**基準:**
- 120〜160文字
- 「この記事を読むと何がわかるか」が明確
- 数字・具体性を入れる（例: 「全国100チーム以上を掲載」）

### 3. 内部リンクの強化（1時間）

**やること:**
- 記事同士をつなぐ内部リンクを増やす
- 特にPVの多いページ（大会情報、チーム紹介）から他の記事へリンク
- SWELLの「関連記事ブロック」「ブログカード」を活用

**なぜ効くか:**
- Googleのクロールが回りやすくなる
- ユーザーの回遊率UP → PV向上 → 広告表示増
- 既存コンテンツの評価向上

## やらなくていいこと

| やらなくていいこと | 理由 |
|-------------------|------|
| 被リンク営業 | ニッチジャンルでは自然リンクで十分。時間対効果が悪い |
| 技術的SEO（Core Web Vitals最適化） | SWELLは元から高速。ConoHaも問題なし。優先度低い |
| 大量の新記事投入 | リライト・既存記事の最適化の方が効く |
| SNSを減らして検索に注力 | Xが最大チャネルなので、SNSを維持しつつ検索を補強が正解 |
| 有料SEOツール | 月3,000セッション規模では不要。Search Console で十分 |

## リライトに関する注意点

### リライト後の影響と対処

- Googleがリライト後の内容を再評価するまで **2〜4週間** かかることがある
- この期間は順位が一時的に下がることがある（正常な挙動）
- **対処法:** 焦って再リライトしない。2〜4週間待って順位が戻るか確認

### リライトのベストプラクティス

- 一度に大量リライトしない → 5記事ずつくらいで様子を見る
- リライト前の順位・CTRをメモしておく
- URL（パーマリンク）は変更しない
