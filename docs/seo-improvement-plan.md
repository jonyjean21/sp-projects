### 下落の要因分析

| 要因 | 影響度 | 根拠 |

| 日数差（23日 vs 31日） | 高 | 約25%の差が日数だけで説明できる |

### 重要な気づき

- **X（SNS）が最大の集客源**であり、検索流入はもともと全体の3割程度
- ニッチジャンル（モルック）のメディアであり、検索ボリューム自体が限定的
- **検索流入の回復 + SNS流入のさらなる拡大**の二軸で考える

## SWELLテーマでのSEO最適化

### SWELL の SEO 設計思想

**重要: SWELLはテーマ本体にメタタグ出力機能を意図的に搭載していない。**
テーマ乗り換え時にSEO設定が消えるリスクを回避するため。
→ **SEO SIMPLE PACK が必須**（これがないとメタタグが一切出力されない）

### SWELL設定で確認すべき項目

| 設定項目 | 場所 | やること |
|---------|------|---------|
| 構造化データ | SWELL設定 > 構造化データ | 運営組織名、URL、ロゴ画像を登録 → JSON-LDで自動生成 |
| 高速化 | SWELL設定 > 高速化 | キャッシュ機能ON、CSS/JS最適化ON、Lazy Load ON |
| 目次 | SWELL設定 | 標準搭載の目次機能が有効か確認 |

### 必須・推奨プラグイン

| プラグイン | 必須度 | 備考 |
|-----------|--------|------|
| **SEO SIMPLE PACK** | **必須** | SWELL開発者（了さん）製。メタタグ、OGP、noindex設定、GA/SC連携 |
| **XML Sitemap & Google News** | 推奨 | サイトマップ生成。SEO SIMPLE PACKにはこの機能がないため併用 |
| WP Multibyte Patch | 推奨 | 日本語WP環境の文字化け防止 |
| XO Security | 推奨 | セキュリティ対策 |

### 入れてはいけないプラグイン（SWELL公式非推奨）

| プラグイン | 理由 |
|-----------|------|
| All in One SEO / Yoast SEO | SEO SIMPLE PACKと競合。メタタグ二重出力でスパム判定リスク |
| Autoptimize | SWELL開発者が「不具合発生の可能性が高い」と明言 |
| Async JavaScript / Scripts-To-Footer | SWELL本体の高速化機能と競合 |
| WP Super Cache / W3 Total Cache | SWELL本体のキャッシュ機能と競合 |
| Lazy Load系プラグイン | SWELL標準のLazy Loadと競合 |
| Table of Contents Plus | SWELL に目次機能が標準搭載 |
| Classic Editor | SWELLはブロックエディター前提の設計 |
| SiteGuard WP Plugin | PHP環境によりログイン不能になる不具合報告あり |
| EWWW Image Optimizer | 常時有効化で画像アップロード不能の報告あり |

> プラグインは全体で7〜10個程度に抑えるのがSWELLのベストプラクティス

## PC時間が少ない人向け 優先施策 TOP3

### 1. Search Console で「お宝キーワード」を発掘（月2〜3時間）

**やること:**
1. Search Console → 検索パフォーマンス → クエリタブ
2. 「表示回数が多いのにクリック率(CTR)が低い」クエリを探す
3. そのクエリに対応する記事のタイトル・meta descriptionを改善（SEO SIMPLE PACKの記事編集画面下部から設定）

**なぜ効くか:**
- すでにGoogleに表示されているが、クリックされていない = タイトル改善だけで流入増
- 新記事を書くより圧倒的に工数が少ない

**具体例:**
- 「モルック ルール」で表示回数100、CTR 3% → タイトルを「【2025年最新】モルックのルールを完全解説」に変更
- → CTR 3% → 8% になれば、それだけで流入が2.7倍

### 2. 構造化データとリッチリザルトの活用（1回3〜4時間で完了）

**やること:**
1. SWELL設定 > 構造化データに運営組織情報を登録（未設定の場合）
2. ルール解説ページに **SWELLのFAQブロック** を追加 → Q&Aリッチリザルトが検索結果に表示
3. 正規品ガイドに **商品レビューブロック** を追加 → 星評価が検索結果に表示
4. 全ページのmeta descriptionが設定されているか確認

**なぜ効くか:**
- リッチリザルト（星評価やQ&A表示）が出ると CTR が大幅に向上
- SWELL標準のブロックを使うだけでJSON-LD構造化データが自動出力される

### 3. 大会情報ページの鮮度維持（月1〜2時間）

**やること:**
- 終了した大会を「結果」セクションに移動、今後の大会を目立つ位置に
- 記事タイトルに「【2025年最新】」「【毎月更新】」などの鮮度シグナル
- 記事の更新日を明示（SWELL の更新日表示設定）
- Xで大会情報の更新を告知 → ソーシャル流入も取る

**なぜ効くか:**
- 「モルック大会情報」は最もPVが多いページ（-41.3%の下落だが依然トップ）
- 大会情報は鮮度が命。最新に保つだけで検索評価が回復する可能性

## キーワード戦略

### 3層のキーワード構造

**第1層: ビッグキーワード（既存記事で維持）**
- 「モルック」「モルック ルール」「モルック 大会」
- 現在上位を維持しているはず → 落とさないことが最優先

**第2層: ミドルキーワード（積極的に狙う）**
- 「モルック 購入」「モルック 正規品」「モルック 遊び方」
- 「モルック チーム」「モルック 練習」「モルック 戦略」
- 「モルック 大会 2025」「モルック 大会 〇〇（地域名）」

**第3層: ロングテール（少ない工数で上位が狙える）**
- 「モルック 初心者 始め方」「モルック 何人で遊べる」
- 「モルック 室内 できる」「モルック 雨の日」
- 「モルック スコア 計算」「モルック 50点 コツ」
- 「モルック 子供 何歳から」「モルック vs ペタンク 違い」

### キーワード調査ツール（無料）

- **ラッコキーワード** (https://rakkokeyword.com) — 「モルック」のサジェスト・関連語を網羅取得
- **Search Console** — 既に表示されているがクリックされていないキーワードの発掘（最も工数対効果が高い）

## コンテンツクラスター戦略

既存のサイト構成は、すでにクラスター構造の素地ができている。これを内部リンクで明確に結びつける。

```
[ピラーページ] トップページ or モルック完全ガイド
    |
    ├── [クラスター] ルール・遊び方
    ├── [クラスター] 大会情報（年度別・地域別）
    ├── [クラスター] チーム紹介
    ├── [クラスター] 正規品購入ガイド
    ├── [クラスター] ツール（スコア計算等）
    └── [クラスター] 動画・YouTube
```

各クラスター↔ピラーページを相互に内部リンク → サイト全体の「モルック」トピックオーソリティが向上。

ニッチメディアはここが最大の武器。

| 要素 | やること |
|------|---------|
| **Experience（経験）** | 大会参加レポート、実際にプレイしている写真・動画、一次情報を掲載 |

| **Authoritativeness（権威性）** | 日本モルック協会等からの被リンク獲得。Xでのソーシャルシグナル活用 |
| **Trustworthiness（信頼性）** | 運営者情報、プライバシーポリシー、情報の出典明記 |

## Xの成長を検索にも波及させる

Organic Social が唯一の成長チャネル（+13.1%）→ これを検索流入にも活かす

- **バズった話題をすかさず記事化** → ソーシャルトラフィック → 検索トラフィックに転換
- **大会結果速報** → Xで拡散 → 記事に誘導 → 検索にもインデックス → 好循環
- **ブランド検索の増加** → Xでの認知拡大により「molkky-hub」「モルックハブ」検索が増える → SEOに好影響

## やらなくていいこと

| やらなくていいこと | 理由 |
|-------------------|------|
| 新しいSEOプラグイン導入 | SEO SIMPLE PACKで十分。他を入れると競合して逆効果 |
| 高速化プラグイン追加 | SWELL本体の高速化機能で完結。追加は不具合の原因 |
| 被リンク営業 | ニッチジャンルでは自然リンクで十分。時間対効果が悪い |
| 大量の新記事投入 | リライト・既存記事の最適化の方が効く。新規は月1〜2本で十分 |
| デザインの大幅リニューアル | SWELLのデザインは十分優秀。SEOに直接影響しない |
| 有料SEOツール | Search Console + ラッコキーワード（無料）で十分 |
| 日々の順位チェック | 週1回のSearch Console確認で十分。日々の変動に一喜一憂しない |

## リライトに関する注意点

### リライト後の影響と対処

1. Googleがリライト後の内容を再評価するまで **2〜4週間** かかる
2. この期間は順位が一時的に下がることがある（正常な挙動）
3. **焦って再リライトしない** → 4週間待って様子を見る
4. 1ヶ月以上改善しなければ、元に戻すことも検討
5. Search Console > URL検査 > 「インデックス登録をリクエスト」で再クロール促進

### リライトのベストプラクティス

- 一度に大量リライトしない → 5記事ずつくらいで様子を見る
- リライト前の順位・CTRをメモしておく
- URL（パーマリンク）は変更しない
- 見出しから主要キーワード（「モルック」等）を外さない
