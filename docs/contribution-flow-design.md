# 寄稿プロジェクト フロー設計メモ

> 作成日: 2026-02-24

## 現状の課題

- 連絡手段が寄稿者ごとにバラバラ（LINE、メール等）
- 入稿→WordPress転記が手動 → PC時間が取れないとボトルネックに
- 寄稿者が今後増える見込み → 今のうちにフローを整備したい

## 選択肢の比較

| 方法 | 費用 | 寄稿者の負担 | 孔雀の手間 | 自動化拡張性 | 判定 |
|------|------|-------------|-----------|-------------|------|
| **Googleフォーム + GAS** | 無料 | 最小（URL1つ） | Phase2で激減 | 高（GAS→WP REST API） | **推奨** |
| WPフォームプラグイン | 年$49〜199 | 低 | 低（直接下書き保存） | 中 | 次善策 |
| Notion | 無料 | やや高（アカウント必要） | WP転記が残る | 中 | 管理用途なら○ |
| LINE公式 | 無料〜 | 最小 | 高（全て手動転記） | 低 | 非推奨 |
| メール | 無料 | 低 | 高（管理が煩雑） | 低 | 非推奨 |

## 推奨フロー: 段階的導入

### Phase 1（すぐやる・30分〜1時間）— Googleフォームで受付統一

```
寄稿者 → Googleフォーム → スプレッドシートに自動記録 → メール通知 → 孔雀が手動でWP投稿
```

**フォーム項目:**
- 寄稿者名（プルダウン or 自由入力）
- 記事タイトル
- 記事カテゴリ（プルダウン：コラム / 大会レポート / チーム紹介 等）
- 本文（長文テキスト）
- 画像（ファイルアップロード、複数可）
- 画像の説明・挿入位置メモ（自由記述）
- 掲載希望日（任意）

**これだけで得られる効果:**
- 連絡手段バラバラ問題が解消
- スプレッドシートで入稿状況を一覧管理
- スマホで確認可能
- フォームの項目で構造を強制 → フォーマット逸脱防止

### Phase 2（余裕ができたら・2〜4時間）— GAS連携で半自動化

```
寄稿者 → Googleフォーム → スプシ → GAS → WP REST APIで下書き自動作成 → LINE Notify通知 → 孔雀はスマホで確認→公開
```

**技術ポイント:**
- `POST /wp-json/wp/v2/posts` で下書き投稿を自動作成
- `POST /wp-json/wp/v2/media` で画像アップロード
- 認証: WordPress標準のアプリケーションパスワード（WP 5.6+）
- LINE Notify で「○○さんから寄稿が届きました。下書きに保存済みです」と通知

**効果:**
- 手動転記がゼロに
- 1記事あたりの作業: 15〜30分 → 5分以下
- スマホだけで確認・公開が完結

### Phase 3（寄稿者10名超になったら）— ステータス管理

- スプレッドシートにステータス列追加（受付済 / 確認中 / 公開済 / 差し戻し）
- GASで公開時にステータス自動更新

## 推奨の理由

1. **寄稿者の負担が最小** — Googleフォーム1つで統一、誰でも使える
2. **孔雀のPC時間を最小化** — Phase 2でスマホだけで完結
3. **段階的に導入可能** — Phase 1は30分で効果が出る
4. **費用ゼロ** — Googleフォーム・GAS・WP REST API全て無料
5. **SWELL互換** — REST APIで下書き → SWELLブロックエディタで仕上げ
