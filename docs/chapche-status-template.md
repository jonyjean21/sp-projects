# チャプチェ会 ステータスレポート テンプレート

> **使い方**: チャプチェ会の前にAIに以下のように頼む
>
> 「チャプチェ会のステータスレポートを作って」
>
> AIがこのテンプレートと `app-status.md`、`tasks/current.md` を参照して
> 報告用レポートを自動生成する。

---

## レポートフォーマット

```markdown
# チャプチェ会 ステータスレポート（YYYY-MM-DD）

## 前回からの変化
- （前回の会議後に起きたこと・進んだことを箇条書き）

## アプリ開発の状況

### レビュー待ち
| アプリ | 状態 | 次にやること |
|--------|------|------------|
| （該当アプリ） | （詳細） | （具体的なアクション） |

### 開発中
| アプリ | 状態 | 次にやること |
|--------|------|------------|
| （該当アプリ） | （詳細） | （具体的なアクション） |

### 要確認（Manusのみ）
- （未確認アプリの一覧と確認事項）

## タスク進捗
- 完了: （前回から完了したタスク）
- 進行中: （今やってること）
- ブロック中: （何かで止まってるもの）

## 今日話したいこと
- （検討ボックスの中身 or 判断が必要なこと）

## 決めたいこと（要判断）
- [ ] （会議中に決めるべきこと）
- [ ] （会議中に決めるべきこと）
```

---

## AIへの指示例

### 会議前（ダッシュボード更新）
```
チャプチェ会のダッシュボードを更新して。
docs/chapche-dashboard.html の中身を最新の状態に書き換えて。
app-status.md と tasks/current.md の情報をもとに。
```
→ ブラウザで `docs/chapche-dashboard.html` を開いてGoogle Meetで画面共有

### 会議前（テキストレポート）
```
チャプチェ会のステータスレポートを作って。
projects/molkky-dome/app-status.md と tasks/current.md を見て。
```

### 会議後
```
チャプチェ会が終わった。以下が決まったこと：
- （箇条書きで決定事項）
- （箇条書きで新しいタスク）

app-status.md と tasks/current.md を更新して。
```

### 中さんが新しいアプリを作ったとき
```
中さんが新しいアプリ「〇〇」を作った。
URL: https://xxx.manus.space
app-status.md に追加して。
```

### コードレビュー後
```
oshi-katsu のコードレビューが完了した。
指摘事項：
- （箇条書き）
app-status.md のレビューログを更新して。
```

## 会議の進め方（最適化版）

### Before（今まで）
```
Notion開く → やることリスト上から順に確認 → 各自の報告 → 検討ボックス確認
→ 結果: 報告で時間の大半が終わる
```

### After（最適化後）
```
1. [2分] ダッシュボード（HTML）を画面共有 → 全員が現状を把握（報告不要）
2. [残り全部] 判断・議論に集中
   - アジェンダの「要判断」項目を順に消化
   - ブロックされてるタスクの解消
   - 新しいアイデアの検討
3. [最後2分] 決まったことの確認 → AI（or SP）がファイル更新
```

**ポイント: 「報告」を「確認」に変える。話すのは判断が必要なことだけ。**
