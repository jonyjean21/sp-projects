Components — 登場するツール
⚡
GAS
Google Apps Script
Googleが提供するサーバーレス実行環境。JavaScriptでスクリプトを書き、時間起動などのトリガーで自動実行できる。無料。
このパイプラインでの役割：
7つのサイトを巡回して記事ネタを収集し、Firebase に登録する
🗄️
Firebase RTDB
Google のリアルタイムDB
Googleのクラウドデータベース。JSON形式でデータを保存・読み書きできる。REST APIで誰でも簡単にアクセス可能。
このパイプラインでの役割：
収集ネタのキュー管理。pending→postedで処理状態を追跡
⚙️
ConoHa PHP cron
レンタルサーバー内の自動実行
ConoHa WINGのサーバー内でPHPスクリプトを定刻に自動実行する機能。サーバー内部から動くのでIP制限を受けない。
このパイプラインでの役割：
パイプラインの心臓部。毎朝7時にFirebase→Gemini→WP投稿を実行
✨
Gemini API
Google の生成AI
GoogleのAIモデル。テキスト生成・翻訳・要約などをAPIで呼び出せる。GPT-4と同等の能力を持ち、大量トークン処理が得意。
このパイプラインでの役割：
収集した記事ネタをスコアリング・翻訳し、日本語の記事として仕上げる
🖼️
Pexels API
フリー写真素材サービス
著作権フリーの写真を提供するサービス。APIで検索・取得が可能。商用利用無料。WordPressのメディアとして自動アップロードできる。
このパイプラインでの役割：
記事のアイキャッチ画像をキーワードで自動検索して設定する
🌐
WordPress REST API
WPの外部操作インターフェース
WordPressが標準で持つAPI。外部プログラムから記事の投稿・更新・取得などを認証付きで操作できる。プラグイン不要。
このパイプラインでの役割：
PHPスクリプトからWPに記事・アイキャッチを自動投稿する
Flow — 実際の処理の流れ
1
GAS が7ソースを巡回・収集
6時間ごと Reddit / HN / Zenn / Qiita / dev.to / GitHub
各ソースの新着記事を取得し、スコアリング（海外+50・コードあり+20）してFirebaseのキューにpendingで登録
2
Firebase がネタをキューとして保持
pending → posted / skipped
収集と投稿を切り離すハブ。処理済みはpostedに更新し重複投稿を防止。ネタが溜まっていれば翌日以降も処理可能
3
ConoHa PHP cron が毎朝7時に起動
毎朝 7:00 JST ★ 正式パイプライン
サーバー内のPHPスクリプトが自動実行。Firebaseからpendingを取得し、スコア順に処理。スラッグ照合で既存記事との重複チェックも実施
4
Gemini API が記事を生成
翻訳 / 要約 / 編集部コメント生成
高スコア1本をピックアップして要約＋独自考察。残りを「その他注目」として要点まとめ。最後に編集部コメントをAIが生成してまとめる
5
WordPress に記事・アイキャッチを自動投稿
Pexels アイキャッチ付き カテゴリ・タグ自動設定
Pexelsからアイキャッチ画像を取得してWPにアップロード。記事本文・タイトル・カテゴリとセットでREST APIから投稿。buildhub.jpに自動公開される
Why — なぜ ConoHa PHP cron なのか
✕ 廃止
GitHub Actions
GitHubが提供するCI/CDツール。リポジトリのイベントやスケジュールでスクリプトを自動実行できる。
GitHub = Microsoft Azure のサーバーから実行される
ConoHa WING は Microsoft / Google の IP を全ブロック
WP REST API に届かず 403 エラー → 構造的に不可
○ 採用
ConoHa PHP cron
ConoHa WINGが提供するcron機能。サーバー内のスクリプトを指定時刻に自動実行できる。
サーバー内部からの実行 → IP制限を受けない
wp-load.php を直接読み込めるため高速・安定
毎朝7時に安定稼働中（アイキャッチ付きで実績あり）
Learnings
01
レンタルサーバーの自動化は「サーバー内 cron」が最強。外部サービスからのAPI呼び出しはIPブロックのリスクがある。
02
収集と投稿をFirebaseで疎結合にすると、どちらかが止まっても影響しない。ネタは翌日以降に処理できる。
03
GAS → Firebase → PHP cron の三層構造が ConoHa WING 環境での最適解。各レイヤーが独立して動く。
