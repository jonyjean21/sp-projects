# SP Portal / モルハブポータル 再構成設計書

## 現状の問題

SP Portalは「SPの全てを管理する場所」を目指しているが、実態はモルハブ管理機能（記事・大会・Xポスト）がメインで、SP個人のプロダクト管理やビルド履歴は付け足し感がある。

さらに、モルハブ関連だけでも以下のツールが分散している：
- **SP Portal**: 記事管理、大会キュー、大会データ、Xポスト、アナリティクス
- **hub-dashboard**: 向上委員会ダッシュボード（KPI表示）
- **molkky-admin**: レガシー管理パネル
- **naimol-admin**: 大会結果→WP投稿
- **championship-admin**: 日本選手権トラッカー管理

また大会セクションが「大会情報」（キュー処理）と「大会データ」（WPビューア）に分裂している。

---

## 再構成方針

### SP Portal = SPの全て
- プロダクトカタログ（appsカタログ統合）
- ビルド履歴
- 全プロジェクト横断ダッシュボード
- 各ポータルへのハブ（リンク集）
- 個人ツール（家計管理等）

### モルハブポータル = モルハブの全て（新規・独立）
- 記事管理
- 大会管理（キュー + 全データ統合）
- Xポスト生成
- アナリティクス（GA4/SC連携）
- パートナー管理
- 大会結果（ナイモル）
- 選手権トラッカー
- 自動化パイプライン状態

### 独立を維持するもの
- **MACHAPポータル**: チーム3人共有。SP専用ではない
- **めぐり帳**: 完全に個人用
- **レシピBOX**: 家族用
- **パートナーLP / partner-portal**: 外部向け
- **公開ページ群**: championship/, viisi-master/ 等

```

Portals（ハブ）
```

2. **プロダクト一覧**: カテゴリ別グリッド（現在のapps/index.html相当）
   - ステータスバッジ（Live/Dev/Archive）

3. **ビルド履歴**: タイムライン形式（最新が上）
4. **クイックリンク**: 各ポータル・外部サービスへのショートカット
5. **最近のアクティビティ**: WP最新投稿

### 大会管理 — 統合設計（最重要）

**現状の問題**: 「キュー処理」と「データ閲覧」が別セクション

**統合UI設計**:

```
┌─────────────────────────────────────────────┐
│ 大会管理                          [手動追加] │
├─────────────────────────────────────────────┤
│ タブ: [全件] [未処理] [要確認] [最近の投稿]  │
├─────────────────────────────────────────────┤
│ 検索: [________] 地域:[▼] 月:[▼] 状態:[▼]  │
├─────────────────────────────────────────────┤
│ ┌───────────────────────────────────────┐   │
│ │ 🟡 PENDING  モルック大会 in 東京       │   │
│ │ 2026/04/15 東京 | IFTTT | → [処理]   │   │
│ └───────────────────────────────────────┘   │
│ ┌───────────────────────────────────────┐   │
│ │ ⚠️ WARNING  モルック体験会 千葉        │   │
│ │ 2026/03/20 千葉 | 公開中 | 規模なし    │   │
│ └───────────────────────────────────────┘   │
│ ┌───────────────────────────────────────┐   │
│ │ ✅ OK  第5回〇〇モルック大会            │   │
│ │ 2026/05/01 大阪 | 公開中 | 50チーム     │   │
│ └───────────────────────────────────────┘   │
│                                             │
│ ← 1 2 3 ... 28 →                           │
└─────────────────────────────────────────────┘
```

**タブの意味**:
- **全件**: WP投稿済み全大会 + Firebase未処理キュー = 全てがここに
- **未処理**: Firebaseキューの pending のみ（IFTTT/GAS/手動）
- **要確認**: 自動検証でwarningが付いた大会
- **最近の投稿**: 直近7日間の投稿

**カード展開（行タップ）**:
- 全ACFフィールドのインライン編集
- ソース表示（IFTTT/GAS/手動、元URL/ツイート）
- 検証結果（開催日なし・都道府県なし・モルック不含・重複疑い）
- WP管理画面リンク
- 保存ボタン（POST で WP更新）

**手動追加**:
- URLペースト → Gemini抽出 → フォーム自動入力 → 保存
- 空フォームからの手入力も可

**統計サイドパネル or 上部**:
- 月別投稿数の棒グラフ
- 都道府県別ヒートマップ
- ソース別割合（IFTTT/GAS/手動）

## 移行計画

### Phase 1: モルハブポータル構築
1. `molkky-portal/index.html` 新規作成
2. SP Portalからモルハブ機能を移植（記事・大会・Xポスト・アナリティクス）
3. 大会セクションを統合版に再設計
4. hub-dashboard の KPI表示を統合

### Phase 2: SP Portal軽量化
1. モルハブ機能を削除
2. ダッシュボードをプロダクトカタログ + ビルド履歴中心に再構成
3. 各ポータルへのハブリンク追加

### Phase 3: レガシー整理
1. molkky-admin → モルハブポータルに吸収（廃止）
2. hub-dashboard → モルハブポータルに吸収（廃止 or リダイレクト）
3. apps/index.html → SP Portalのダッシュボードが代替（廃止 or リダイレクト）

## 技術方針
- 全て単一HTML（GitHub Pages、ビルド不要）
- localStorage永続化（WP認証は共有キーで横断）
- WP REST API（SiteGuard WAF対策: POSTのみ）
- Firebase RTDB（キュー・チャプ会・選手権）
- Vercel API Proxy（Gemini・Drive）
