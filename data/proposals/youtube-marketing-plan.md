# YouTube Analytics Dashboard 改善計画書
## マーケター視点のチャンネル成長戦略 + ダッシュボード進化プラン

**作成日**: 2026-03-05
**対象プロダクト**: youtube/index.html（MACHAPプロダクト）
**最初のターゲット**: @nan_yaku_CH（何の役にも立たないCH）

## 現状分析: 今のダッシュボードの強みと弱み

### 既に実装済みの機能（v1.0の到達点）

| タブ | 機能 | 評価 |
|------|------|------|
| Overview | KPI3枚（総再生・登録者・動画数）、再生数推移チャート、Top3動画、エンゲージメント概要 | 基本は押さえている |
| Videos | 動画一覧（ソート3種・ページネーション・サムネイル表示） | 実用的 |
| Patterns | 曜日別・時間帯別・タイトル文字数vs再生数・月別推移 | 分析の種がある |
| Radar | 6軸レーダーチャート + スコアバー + 改善ポイント自動生成 | 方向性は良い |

### インフラ構成

- **Vercel Proxy**: `vercel-api/api/youtube-proxy.js` が YouTube Data API v3 をプロキシ
  - 対応エンドポイント: `channels`, `search`, `videos` の3種のみ
  - YouTube API Key は Vercel 環境変数で管理（キー非公開化済み）
- **接続方式**: チャンネルURL or @ハンドル入力 → Vercel Proxy経由でデータ取得
- **キャッシュ**: sessionStorage（セッション中は再呼び出しなし）
- **デモモード**: 50動画分のモックデータで全機能体験可能

### 致命的な弱点（ユーザーフィードバックから）

1. **スマホ表示が弱い** — レスポンシブ対応はしているが、KPIカードが1列になるだけ。Chart.jsのグラフがスマホで小さすぎて読めないケースあり
2. **曜日別分析の誤解を生む構造** — 「何曜日に投稿したか」と「何曜日に視聴されたか」が混在。投稿曜日×投稿後の視聴推移を分けないと、「土曜に投稿したから土曜の数字が高い」だけのトートロジーになる
3. **アルゴリズムの視点がゼロ** — YouTubeがなぜその動画を推奨するのか、何を改善すればアルゴリズムに乗るのかの情報がない
4. **次のアクションが分からない** — データは見えるが「だから何をすべきか」が弱い。レーダータブの改善ポイントはハードコードされた汎用文言
5. **CTR/インプレッション/平均視聴時間がない** — YouTube Data API v3 では取得不可（YouTube Analytics API + OAuth2.0が必要）。これらはYouTube成長の最重要指標

### 1-2. Overview タブ改善

#### KPIカードの拡張（3枚 → 6枚、2行グリッド）

```
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 総再生数    │ │ 登録者数    │ │ 総動画数    │
│  12,345   │ │   523    │ │    48    │
│ +15% ↑   │ │ — (※)   │ │ +3 今月  │
└──────────┘ └──────────┘ └──────────┘
┌──────────┐ ┌──────────┐ ┌──────────┐
│ 平均再生数   │ │ 投稿ペース   │ │ 成長スコア   │
│    856    │ │ 3.2本/月  │ │  68/100  │
│ vs前期比   │ │ ↑改善中   │ │ Bランク   │
└──────────┘ └──────────┘ └──────────┘
```

**新規追加KPI**:
- **平均再生数**: 直近10本の平均。前10本との比較で上昇/下降を表示
- **投稿ペース**: 直近3ヶ月の月あたり投稿数。「週1→週2で成長加速」のような具体的提案付き
- **成長スコア**: 総合評価をA-Fランクで表示。一目で「自分のチャンネルの状態」がわかる

#### 再生数推移チャートの改善

- 30動画ではなく **全動画の時系列** で折れ線表示（期間フィルタ: 直近30日/90日/全期間）
- **移動平均線**（5本移動平均）を重ねて、トレンドを可視化
- ホバー時に動画タイトル + 投稿曜日 + 時間帯を表示

#### トップ動画の改善

- Top3 → **Top5に拡張**
- 各動画に「なぜ伸びたか」の推定理由を1行で表示
  - 例: 「タイトルにトレンドワードを含む」「平均より長い動画で視聴維持が高い可能性」

#### スマホ対応の改善

```css
/* KPIカード: スマホでは2列 x 3行（現在は1列6行で長すぎる） */
@media (max-width: 768px) {
  .kpi-grid {
    grid-template-columns: repeat(2, 1fr);
    gap: 8px;
  }
  .kpi-value { font-size: 20px; }
  .kpi-label { font-size: 10px; }
}
```

- KPIカードをスマホで2列表示（1列だと縦に長すぎてスクロールが面倒）
- チャートのaspectRatioをスマホ時に1.5に変更（横長すぎて読めない問題の解消）
- タップ領域を44px以上に統一（モバイルUXガイドライン準拠）
- ボトムナビのアイコンサイズを24pxに拡大

### 1-3. Videos タブ改善

#### 動画カードの情報追加

```
┌────────────────────────────────────────┐
│ ┌──────┐  動画タイトル                    │
│ │サムネ │  2026/03/01(土) 18:00 公開       │  ← 曜日・時間を追加

│           [YouTubeで見る →]              │
└────────────────────────────────────────┘
```

**追加データ**:
- **公開曜日・時間**: 投稿タイミングの傾向を視覚的に把握
- **平均比較**: その動画がチャンネル平均の何倍かを表示（1.5倍なら緑、0.5倍なら赤）
- **タグ**: YouTube Data API v3 の `videos` エンドポイントで `part=snippet` を追加すれば `tags` が取得可能

#### フィルタ機能の追加

- **期間フィルタ**: 直近30日/90日/1年/全期間
- **再生数フィルタ**: 1000以上/500以上/全て
- **検索**: タイトルのキーワード検索

#### Vercel Proxyの拡張が必要な点

現在の `youtube-proxy.js` は `part` パラメータを透過しているので、`part=snippet,statistics,contentDetails` にするだけで **タグ・説明文** も取得可能。追加のプロキシ改修は不要。

### 1-4. Timing タブ（旧Patterns の進化版）

**核心的な改善: 「投稿タイミング × 視聴データ」のクロス分析**

現在の曜日別/時間帯別チャートは「投稿された曜日」ごとの平均再生数を出しているだけ。これでは「水曜に投稿した動画が多かったから水曜の合計が多い」という偏りを生む。

#### 新機能 1: ヒートマップ（曜日 x 時間帯）

```
         0-3  3-6  6-9  9-12  12-15  15-18  18-21  21-24
月曜     ░░   ░░   ░░   ██    ░░     ░░     ████   ██
火曜     ░░   ░░   ░░   ░░    ██     ░░     ██     ░░
水曜     ░░   ░░   ██   ░░    ░░     ██     ████   ██
...
日曜     ░░   ░░   ░░   ████  ██     ░░     ██████ ████
```

- Chart.jsのmatrixプラグイン（`chartjs-chart-matrix`）で実装可能
- CDN: `https://cdn.jsdelivr.net/npm/chartjs-chart-matrix@2`
- 色の濃さ = 平均再生数の高さ
- **最適投稿タイミングをハイライト**: 「日曜18-21時が最も再生数が高い」

#### 新機能 2: 投稿後72時間の伸び分析

YouTube Data API v3 単体では72時間ごとの再生推移は取れないが、**定期的にデータを取得してlocalStorageに蓄積** することで疑似的に実現可能。

```
仕組み:
1. ダッシュボードを開くたびに全動画の現在の再生数を取得
2. 前回取得時のデータとの差分を計算
3. 「この動画は公開後X日でY回再生増加」を推定
4. localStorageに履歴として蓄積
```

これにより「公開後48時間でどれだけ伸びるか」のトレンドが見えるようになる。

#### 新機能 3: 投稿間隔 vs パフォーマンス

```
前回の投稿からの間隔（日数）  →  再生数
3日後に投稿: 平均 1,200回
7日後に投稿: 平均 950回
14日以上: 平均 600回

→ インサイト: 「3-5日おきの投稿がこのチャンネルでは最もパフォーマンスが高い」
```

- 「投稿間隔が空くとアルゴリズムがチャンネルを忘れる」という仮説を検証可能

#### 新機能 4: タイトル分析の深掘り

- 文字数だけでなく **キーワード頻出分析** を追加
- 【】や絵文字の使用有無 × 再生数の相関
- 疑問形タイトル vs 断定形タイトルの比較

YouTubeアルゴリズムが重視する指標に基づいたヘルスチェック。

```
┌─────────────────────────────────────────┐
│ アルゴリズムヘルスチェック                    │
│                                           │
│ ✅ エンゲージメント率: 3.2%（良好）           │
│    → 業界平均2-4%の範囲内                    │

│                                           │
│ ❌ タイトル最適化: スコア 35/100              │
│    → キーワードが不足。検索されやすい          │
│      ワードを含めましょう                    │
│                                           │
│ ⚠️ サムネイル多様性: 中程度                   │
│    → テキスト付きサムネイルの比率が低い        │
│                                           │
│ ✅ コンテンツ一貫性: 高い                     │

│    → ショートは新規視聴者獲得の最速ルート      │

```

#### ヘルスチェック項目の算出ロジック

| 項目 | 算出方法 | データソース |
|------|---------|-------------|
| エンゲージメント率 | (いいね+コメント) / 再生数 | Data API |
| 投稿頻度 | 直近3ヶ月の月あたり投稿数 | Data API |
| タイトル最適化 | 文字数20-40字 + 【】含有 + 数字含有 をスコア化 | Data API snippet |
| サムネイル多様性 | サムネイルの色分散（Canvas APIで解析可能） | サムネイル画像 |
| コンテンツ一貫性 | タグの集中度（上位5タグが全タグの何%を占めるか） | Data API tags |
| ショート動画活用 | duration < 60秒 の動画数 | Data API contentDetails |

**重要**: CTR（クリック率）と平均視聴時間は YouTube Analytics API (OAuth2.0) でしか取得できないため、Data API で取得可能な指標で代替指標を構築する。将来的にOAuth対応した場合に拡張可能な設計にしておく。

#### レーダーチャートの改善

6軸を以下に再定義:
1. **コンテンツ力**: 平均再生数のベンチマーク比較
2. **エンゲージメント**: いいね率 + コメント率の複合指標
3. **一貫性**: 投稿間隔の安定度（標準偏差が小さいほど高スコア）
4. **最適化度**: タイトル・タグ・説明文の充実度
5. **成長性**: 直近10本 vs 前10本の再生数成長率
6. **活動量**: 月あたり投稿数のベンチマーク比較

#### 段階的なアプローチ

**Phase 1（現行: APIキーのみで可能な範囲）**:
- サムネイル画像を並べて「クリックしたくなるか」を視覚的にレビュー
- サムネイルの「テキスト有無」「人物の有無」「色使い」を手動チェックリストで評価
- サムネイルと再生数の相関を目視で分析する機能

**Phase 2（将来: OAuth2.0対応時）**:
- YouTube Analytics API を使ってCTR/インプレッション/平均視聴時間を取得
- Vercel Proxy に OAuth2.0 フローを追加（`/api/youtube-analytics-proxy`）
- チャンネルオーナーが自分のアカウントでOAuth認証 → Analytics APIにアクセス
- ダッシュボードにCTRヒートマップ、視聴維持率グラフを追加

**OAuth2.0対応の技術要件**:
- Google Cloud Console で OAuth 同意画面を設定
- Vercel に `YOUTUBE_OAUTH_CLIENT_ID` / `YOUTUBE_OAUTH_CLIENT_SECRET` を追加
- フロントエンドに Google Sign-In ボタンを追加
- アクセストークンを取得 → Vercel Proxy 経由で Analytics API を叩く
- スコープ: `yt-analytics.readonly`

## Part 2: YouTubeマーケティング戦略

### 2-1. YouTubeアルゴリズムの仕組み（2025-2026年最新）

YouTubeのアルゴリズムは「動画のランキングエンジン」ではなく「視聴者のマッチングエンジン」である。これを理解することが全ての出発点。

#### 4つの推薦面

| 推薦面 | アルゴリズムの重視点 | チャンネル成長への影響 |
|--------|-------------------|---------------------|
| **ホームフィード** | 視聴者の過去の視聴パターン、類似ユーザーの行動、動画の初期パフォーマンス | 最大のトラフィックソース。ここに乗れるかが成長の鍵 |
| **検索結果** | タイトル・説明文・タグのキーワード一致、動画の関連性、エンゲージメント | 安定した流入源。SEOが効く |
| **関連動画** | 視聴中の動画との関連性、視聴者の過去の行動 | 既存視聴者の深掘り。再生リストが効果的 |
| **ショートフィード** | 視聴完了率、リプレイ率、エンゲージメント（2026年: 長尺と完全独立） | 新規リーチの最速ルート |

#### 2025-2026年の重要な変更点

1. **満足度 > 視聴時間**: 以前は「視聴時間（Watch Time）」が最重要だったが、2025年以降は「視聴者満足度調査」「繰り返し視聴」「Not Interestedクリック率」が上位シグナルに昇格
2. **ショートと長尺の分離**: ショートのパフォーマンスが長尺のレコメンドに影響しなくなった（逆も同様）。これにより、ショートを気軽に試せるようになった
3. **小規模クリエイターへの配慮**: YouTubeが公式に「良いコンテンツは小さなチャンネルでも推薦する」と明言。チャンネル規模より動画単位のパフォーマンスが重視される
4. **コミュニティ投稿の重要性向上**: コミュニティ投稿へのエンゲージメントが、動画の推薦にも影響するようになった

#### アルゴリズムが見ている主要シグナル

```
最重要:
├── クリック率（CTR）: サムネイルが表示された時にクリックされる確率
├── 平均視聴率: 動画のうちどれだけの割合が視聴されたか
├── 視聴後の行動: 動画を見た後、チャンネル内の別動画を見るか
└── 満足度シグナル: 繰り返し視聴、共有、保存

重要:
├── エンゲージメント: いいね率、コメント率
├── 投稿の一貫性: 定期的な投稿スケジュール
├── セッション貢献: その動画がYouTubeでの視聴セッションを延長するか
└── チャンネルの専門性: 特定トピックへの集中度

補助的:
├── タイトル/説明文/タグ: 検索での発見可能性
├── サムネイルの品質: A/Bテストでの優位性
└── 動画の長さ: ジャンルに適した長さか
```

### 2-2. 登録者を増やすための具体的施策

#### A. サムネイル最適化

YouTubeで最も重要な「1秒の勝負」がサムネイル。CTRが1%上がるだけで推薦の確率が劇的に変わる。

**鉄則**:
1. **テキストは3-5語以内**: スマホで読めるサイズ。文字は太く、背景とコントラストをつける
2. **人物の表情**: 驚き・喜び・疑問の表情は CTR を上げる（40代おじさん3人なら、リアクション芸を前面に）
3. **色のコントラスト**: YouTubeの白/ダーク背景から浮く色を使う。赤・黄色・緑が目立つ
4. **動画内容と一致**: クリックベイトは短期的にCTRを上げるが、満足度シグナルを下げてアルゴリズムに嫌われる
5. **統一感**: チャンネルのサムネイルに一貫したデザインテンプレートを使うと、ブランド認知が上がる

**nan_yaku_CHへの具体的アドバイス**:
- 3人のメンバーの「表情」をサムネイルの武器にする
- モルック投げのダイナミックなシーンを切り出す
- 「おじさんが本気で○○した結果」のようなギャップ系テキスト

#### B. タイトル最適化

**鉄則**:
1. **最初の28文字が勝負**: スマホでは28文字程度しか表示されない
2. **検索キーワードを前半に**: 「モルック 練習」「モルック 投げ方」等を先頭に
3. **数字を使う**: 「3つのコツ」「5分で分かる」は CTR が高い
4. **感情を入れる**: 「衝撃」「まさか」「感動」等の感情ワード
5. **【】の活用**: 日本のYouTubeでは【】が CTR を上げる傾向

**パターン例**:
- 「【モルック】おじさん3人が本気で50点ゲームした結果」
- 「モルック初心者が100日練習したら大会でどうなった？」
- 「【衝撃】モルックの投げ方を変えたら勝率が3倍になった」

#### C. 投稿頻度と一貫性

**最低ライン**: 週1本の長尺動画
**理想**: 週1-2本の長尺 + 週3-5本のショート
**最重要**: 曜日・時間を固定する（視聴者の習慣を作る）

#### D. ショート動画戦略

2026年のYouTubeで小規模チャンネルが成長する最速ルート。

**ショートの特徴**:
- 新規リーチが長尺の5-10倍（アルゴリズムが広く配信する）
- 1,000-5,000登録者帯で平均2,600回再生（TikTok/Reelsを上回る）
- 視聴完了率が最重要指標（15-35秒が最適、60秒は完走率が下がる）
- ショートからの登録者転換率は低い（1%程度）が、「認知」としては絶大

**nan_yaku_CHのショート戦略**:
1. **モルック練習のスーパーショット集**: 50点ぴったり、奇跡の1本抜き等
2. **おじさん3人のリアクション集**: 失敗して叫ぶシーン、喜ぶシーン（15秒）
3. **モルックルール解説**: 「モルックって何？」を30秒で（検索流入狙い）
4. **対戦ダイジェスト**: 3人対戦の1ゲームを30秒にまとめる
5. **ラジオのハイライト切り出し**: トークの面白い部分を15-30秒に

**技術的なポイント**:
- 縦動画（9:16）で撮影
- 最初の1-2秒で視聴者を掴む（テキストオーバーレイ or 衝撃シーン）
- BGMは「トレンドの音源」を使う（発見率アップ）
- ハッシュタグ: #モルック #Shorts #40代の挑戦

### 3-3. 週次レポート機能

#### データの蓄積

```javascript
// ダッシュボードを開くたびにスナップショットを保存
function saveWeeklySnapshot() {
  const snapshots = JSON.parse(localStorage.getItem('yt_dash_snapshots') || '[]');
  const today = new Date().toISOString().split('T')[0];

  // 同日のスナップショットは更新、なければ追加
  const existing = snapshots.findIndex(s => s.date === today);
  const snapshot = {
    date: today,
    subscriberCount: state.channel.subscriberCount,
    viewCount: state.channel.viewCount,
    videoCount: state.channel.videoCount,
    videos: state.videos.map(v => ({
      id: v.id,
      views: v.views,
      likes: v.likes,
      comments: v.comments
    }))
  };

  if (existing >= 0) {
    snapshots[existing] = snapshot;
  } else {
    snapshots.push(snapshot);
  }

  // 直近90日分のみ保持
  const cutoff = new Date();
  cutoff.setDate(cutoff.getDate() - 90);
  const filtered = snapshots.filter(s => new Date(s.date) >= cutoff);

  localStorage.setItem('yt_dash_snapshots', JSON.stringify(filtered));
}
```
