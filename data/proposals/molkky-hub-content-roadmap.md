# モルハブ コンテンツロードマップ 2026

> 作成日: 2026-03-06
> 方針: データベース型メディア（方針B）
> 目標: 競合（jajapatatas）を全面で上回り、日本最大のモルック情報ハブを確立する

## 現状スナップショット

| 指標 | 現在 |
|------|------|
| 大会情報 | 580件（CPT: tournament） |
| チームDB | 70件（CPT: molkky_team） |
| 記事 | 25件（通常投稿） |
| 月間セッション | 約3,000 |
| 自動収集 | 3ルート（GAS RSS / IFTTT X / 手動） |

## Phase 1: 週間コンテンツの自動化（3月中）

### 1-1. 週間大会まとめ記事 ✅ 第1号作成済み（ID:6533）
- **頻度**: 毎週木〜金に公開
- **内容**: 今週末の大会一覧 + 来週以降の注目大会
- **データソース**: WP tournament CPT（ACF event_date でフィルタ）
- **自動化**: Claude Code スキル `/weekly-roundup` を構築
  - tournament CPTから開催日ベースで自動抽出
  - 記事テンプレートに流し込み
  - draft作成 → SP確認 → 公開
- **Xポスト**: 記事公開後に `/promote` で自動生成
- **カテゴリ**: 週間まとめ（ID:139）+ 大会情報（ID:8）

### 1-2. 日次記事の自動生成パイプライン
- **目標**: 1日1記事ペース
- **コンテンツタイプ**（ローテーション）:
  1. **大会プレビュー** — 注目大会の事前紹介（主催者情報、過去実績、参加方法）
  2. **大会レポート/結果速報** — 終了した大会の結果まとめ（X投稿から情報収集）
  3. **チームスポットライト** — 70チームDBから1チームずつ深掘り紹介
  4. **モルック豆知識/コラム** — ルールの細かい話、戦術、道具メンテ等
  5. **週間まとめ** — 上記1-1（週1回）
- **Xポスト経由の情報収集**:
  - IFTTT で収集済みのモルック関連ポストを材料に
  - 大会結果、チーム活動、話題のニュースを記事化
  - 「Xで話題」的な切り口で速報性を出す
- **自動化フロー**:
  - 毎日: キュー確認 → ネタ選定 → 下書き生成 → SP確認 → 公開
  - `/daily-article` スキル構築

### 1-3. Xアカウント運用の強化
- 記事公開のたびに `/promote` でポスト生成
- 大会結果や注目情報を `/repost` でリアルタイム反応
- 目標: 週5ポスト以上

## Phase 2: 構造化コンテンツの充実（3〜4月）

### 2-1. 大会カレンダーページ
- **URL**: /event/calendar/ （新規固定ページ or カスタムページ）
- **機能**:
  - 月別表示（カレンダーUI）
  - 都道府県フィルタ
  - 大会種別フィルタ（オープン/体験会/リーグ戦）
  - WP tournament CPTのACFデータをAPIで取得して表示
- **技術**: 静的HTML + JS（WP REST API）、sp-projectsでホスト or WP固定ページ
- **地図は不要**（ユーザー明言）
- **優先度**: 高（競合の最大の武器がカレンダーアプリ）

### 2-2. 会場情報一覧ページ
- **URL**: /venue/ （新規）
- **内容**:
  - モルックが開催される公園・施設の情報DB
  - 住所、アクセス、地面の種類（芝/土/人工芝）、設備
  - 過去にその会場で開催された大会リスト（tournament CPTから自動リンク）
  - 写真（顔なし風景写真をSPがアップロード）
- **データ構造**: 新しいCPT or ACF + タクソノミー
- **競合との差別化**: jajapatatasには会場情報なし

### 2-3. パートナー記事の公開
- **WP ID:5524**（下書き済み）
- slug: hubpartner
- 内容: data/wp-hubpartner-content.html に準備済み
- 料金: ライト5K/月、スタンダード10K/月、応援ボード20K/月、バナー10K/月、専用記事30K/本
- **タスク**: 最終確認 → 公開 → パートナーLP（partner/）からリンク

## Phase 3: 主要大会ハブ & 特集（4〜5月）

### 3-1. 日本選手権2026 特設ページ
- **URL**: /championship-2026/ （新規固定ページ）
- **内容**:
  - 予選情報（地区ブロック別）
  - 出場チーム/選手リスト
  - 日程・会場
  - 過去の結果アーカイブ
  - トラッカー（championship/ の埋め込み or リンク）
- **データ連携**: championship-admin/ のFirebaseデータを活用
- **競合対策**: jajapatatasの選手権記事（3500+字の詳細記事）を上回る情報量

### 3-2. 主要大会ハブページ群
- **日本オープン** — /japan-open/
- **ビーチモルック** — /beach-molkky/
- **M-Mania** — /m-mania/ （競合の得意領域を侵食）
- 各ページ: 概要、日程、過去結果、参加方法、関連記事リンク

### 3-3. 大会記事へのデフォルトアイキャッチ設定
- 580件のtournament CPTにアイキャッチが未設定
- **方針**: エリア別 or 汎用のデフォルト画像を一括設定
  - 顔なしの風景写真・道具写真をSPがアップロード
  - 都道府県別に割り当て（例: 兵庫→兵庫津ミュージアム周辺写真）
  - WP REST APIで一括更新
- **効果**: アーカイブページの見栄えが劇的に改善

## Phase 4: 記事品質の底上げ（継続）

### 4-1. 写真ストック運用
- **方針**: SPが使える写真をWPメディアにアップロード → Claudeが自動選定
- FileBirdプラグインでフォルダ管理（用途別: 大会風景/道具/会場 等）
- 顔写真NG / 後ろ姿・道具・風景のみ使用可
- 記事作成時に自動でストックから適切な画像を挿入

### 4-2. 既存記事のリッチ化
- 25件の記事に画像追加・内容更新
- 内部リンク強化（大会→チーム→ルール→道具の回遊）
- 構造化データ（FAQ、HowTo）の追加
