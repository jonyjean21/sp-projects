# YouTubeダッシュボード 構想設計書

## 概要

**MACHAPのプロダクトの1つ**として開発する、YouTubeチャンネル分析ダッシュボード。
モルハブとは無関係。SNSダッシュボードを人に届けていくための活動の第一弾。
最初はサプライズプレゼントとして特定の人に贈る。

### 位置づけ
- **所属**: MACHAPプロダクト（5STEPサイクルで管理）
- **対象**: YouTubeチャンネルを持つ誰でも（汎用ツール）
- **形態**: 独立ページ（`youtube/index.html`）— どのポータルにも依存しない
- **将来展開**: Instagram / X 等の他SNSダッシュボードにも横展開可能

## 技術方針

### 元コード（Claude AI生成）について
前セッションで共有されたコードはReact + Recharts製（約650行）。
これをそのまま使うのではなく、**Chart.js + バニラJS**で再構築する。

### なぜChart.js + バニラJSか

| 技術 | 説明 |
|------|------|
| **バニラJS** | フレームワーク（React/Vue等）を使わない素のJavaScript。SPプロジェクト全体の標準。ビルドツール不要、HTMLファイル1つで完結 |
| **Chart.js** | 最も広く使われているチャートライブラリ。CDN1行で導入可能 |

| 選択肢 | メリット | デメリット | 判定 |
|--------|---------|-----------|------|
| **Chart.js + バニラJS** | 軽量・CDN1行・レーダー/棒/折れ線/散布図全対応・アニメーション付き・レスポンシブ | Rechartsほどのカスタマイズ性はない | **採用** |
| React + Recharts | 元コードがそのまま使える・高機能 | React依存（CDN読込で重い）・他プロジェクトと技術が異なる | 不採用 |
| D3.js | 最も自由度が高い | 学習コスト高・コード量が多い | 不採用 |
| CSS only | 依存ゼロ | レーダーチャートが困難・表現力に限界 | 不採用 |

### Chart.jsでできること（元コードの全機能をカバー）
- **折れ線/エリアチャート**: 再生数推移 ✅
- **棒グラフ**: 曜日別パフォーマンス、月別推移 ✅
- **レーダーチャート**: チャンネル強み分析 ✅
- **散布図**: タイトル文字数 vs 再生数 ✅
- **ドーナツ/円グラフ**: カテゴリ比率 ✅
- **アニメーション**: 読み込み時・切替時の滑らかなトランジション ✅
- **レスポンシブ**: スマホ/PC自動対応 ✅
- **ダークテーマ**: CSS変数で完全対応 ✅

### CDN構成
```html
<!-- Chart.js: これ1行だけ -->
<script src="https://cdn.jsdelivr.net/npm/chart.js@4"></script>

<!-- 本体 -->
<script>

  new Chart(ctx, {
    type: 'line',
    data: { ... },
    options: { responsive: true, ... }
  });
</script>
```

## YouTube API

### エンドポイント
```
GET /youtube/v3/channels
  ?part=snippet,statistics
  &id={channelId}
  &key={apiKey}
→ チャンネル名、登録者数、総再生数、総動画数

GET /youtube/v3/search
  ?part=snippet
  &channelId={channelId}
  &maxResults=50
  &order=date
  &type=video
  &key={apiKey}
→ 動画一覧（ID、タイトル、公開日、サムネイル）

GET /youtube/v3/videos
  ?part=statistics,contentDetails
  &id={videoId1,videoId2,...}
  &key={apiKey}
→ 再生数、いいね、コメント数、動画時間
```

### API呼び出し方式
- **クライアントサイド直接呼び出し**（推奨）
- APIキーはlocalStorageに保存（外部サーバーに送信しない）
- YouTube Data API v3は無料枠: 10,000ユニット/日（十分）

### データ永続化
- APIキー: localStorage
- チャンネルID: localStorage
- 取得データキャッシュ: sessionStorage（セッション中はAPI再呼び出し不要）
- 最終取得日時を表示（データの鮮度がわかる）

## デモモード

APIキーがなくても全機能を体験できるモード。

- モックデータ: 架空のチャンネル（50動画分）
- 全タブ・全チャートが動作
- 「これがあなたのチャンネルならこう見える」というイメージを伝えられる
- プレゼント時にまずデモで見せる → APIキーを設定して実データに切替

## 実装ステップ
1. `youtube/index.html` として単一HTMLファイルで構築（Chart.js CDN + バニラJS）
2. デモモードを先に実装（モックデータで4タブ全て動作）
3. YouTube API接続を実装（接続画面 → データ取得 → チャート描画）
4. レスポンシブ対応（PC=サイドバー / スマホ=ボトムナビ）
5. 改善ポイント自動生成ロジック実装
6. MACHAPプロダクトとして登録（MACHAP Portal + apps カタログ）
