# MACHAP Portal — 各STEP機能深掘り計画書

## 基本方針
- **スマホファースト**: チーム3人がスマホで使う前提。PCはサブ
- **入力のハードルを極限まで下げる**: 思いついた瞬間に投稿できる簡単さ
- **自動分類**: 人間が分類を考えなくていい。投稿したら自動でカテゴリ・STEP判定
- **可視化で意思決定を助ける**: 次に何を作るか、3人で選べる画面
- **5STEPが回っている実感**: サイクルの進行を視覚的に体感できる
- **発信は仕組みで回す**: チャネルを意識しない人でも発信できる体制づくり

## 1. FIND — 課題・アイデア発見ボード

### 現状の問題
- 単なるリスト表示で、投稿しやすさ・選びやすさが弱い
- アイデアの質を評価する仕組みが投票ボタンだけ

### 改修案

#### A. クイック投稿 + 自動カテゴライズ（スマホ最適化）
```
┌──────────────────────────────┐
│ 💡 思いついたことを1行で        │
│ [____________________________] │
│                     [投稿 →]  │
└──────────────────────────────┘
        ↓ 投稿すると自動で…
┌──────────────────────────────┐
│ 💡 モルック団体の練習管理を     │
│    一元化できたらいいな        │
│                              │
│ 自動判定:                     │
│  カテゴリ: [モルック] (変更▼)   │
│  STEP:    [FIND] (変更▼)      │
│                              │
│ → 詳細を追記  → そのまま投稿   │
└──────────────────────────────┘
```
- **1行入力で即投稿**。タイトルだけでOK
- **自動カテゴライズ**: 投稿内容からカテゴリを自動判定（Gemini活用）
  - カテゴリ例: モルック / コミュニティ / ビジネス / ツール / その他
  - 間違っていればワンタップで変更可能
- **自動STEP判定**: 内容に応じてどのSTEPに属するか自動振り分け
  - 「〇〇を作りたい」→ FIND
  - 「〇〇が完成した」→ MAKE
  - 「〇〇に提案した」→ OFFER
- LINEに送る感覚で使える軽さ
- 投稿後にカード展開して概要・ターゲットを追記可能

#### B. カンバンビュー（ステータス軸）
```
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ 💡 発見  │  │ 🔍 検討  │  │ ✅ 採用  │  │ 🚀 実行  │
│         │  │         │  │         │  │         │
│ カードA  │  │ カードC  │  │ カードE  │  │         │
│ カードB  │  │ カードD  │  │         │  │         │
│         │  │         │  │         │  │         │
└─────────┘  └─────────┘  └─────────┘  └─────────┘
        ←  スワイプで移動  →
```
- スマホではカードを**長押し→ドラッグ**でステータス変更
- または**スワイプジェスチャー**で右に送る = ステータス進行
- 各列のカード数がひと目でわかる

#### C. カテゴリビュー（別軸）
```
切替: [ステータス別] [カテゴリ別]

┌──────────┐  ┌──────────┐  ┌──────────┐
│ 🎯 モルック │  │ 🤝 コミュ  │  │ 💼 ビジネス │
│    5件    │  │    3件    │  │    2件    │
│           │  │           │  │           │
│ 団体管理   │  │ 地域連携   │  │ 収益化    │
│ 大会運営   │  │ 勉強会    │  │ 法人化    │
│ ルール解説 │  │ コラボ    │  │           │
│ 初心者教室 │  │           │  │           │
│ 動画企画   │  │           │  │           │
└──────────┘  └──────────┘  └──────────┘
```
- ステータス別（発見→検討→採用→実行）とカテゴリ別を**ワンタップ切替**
- カテゴリはカスタム追加可能
- カテゴリ別に見ることで「モルック関連ばかりでコミュニティが手薄」等の偏りを発見
- 各カードにはステータスバッジも表示（どちらのビューでも状態がわかる）

#### D. 評価マトリクス（選びやすさ）
```
           影響度 大
              ↑
    ┌─────────┼─────────┐
    │ 検討枠  │ 即やる！  │
    │         │  ★カードE │
    │─────────┼─────────│
    │ 保留    │ 余裕あれば │
    │         │         │
    └─────────┼─────────┘
              →
           実現容易度 高
```
- アイデアに「影響度」「実現容易度」の2軸スコア（1-5）を設定
- マトリクスに自動プロット
- **右上（高影響×容易）= 次に作るべきもの**が一目瞭然
- 3人で議論しやすい可視化

#### E. アイデアカード詳細
```
┌──────────────────────────┐
│ 💡 モルック団体管理アプリ    │
│ ────────────────────────  │
│ 概要: 各団体の練習日・     │
│   メンバー管理を一元化     │
│                          │
│ カテゴリ: 🎯モルック        │
│ ターゲット: モルック団体代表  │
│ 課題: 練習参加連絡がバラバラ │
│                          │
│ 影響度: ★★★★☆  容易度: ★★★☆☆ │
│                          │
│ [サ✓] [中✓] [師○]  投票: 2  │
│                          │
│ ステータス: [検討中 ▼]      │
│ 関連メモ: チャプ会260303で話題 │
│                          │
│ [編集] [Project化→] [削除]  │
└──────────────────────────┘
```

## 2. MAKE — プロダクト化トラッカー

### 現状の問題
- タスクリストは普通のTODOと変わらない
- 「1週間でプロダクト化」のスピード感が可視化されていない
- 完成したプロダクトの一覧管理がない

### 改修案

#### A. スプリントボード（週次ビュー）
```
┌──────────────────────────────────┐
│ Sprint: 3/3(月) 〜 3/9(日)        │
│ ━━━━━━━━━━━━━━━━━━━━ 60%        │
│                                  │
│ 月 火 水 木 金 土 日              │
│ ── ── ── ██ ░░ ░░ ░░   ← 今日   │
│                                  │
│ [TODO: 3]  [進行中: 2]  [完了: 4] │
└──────────────────────────────────┘
```
- **今日が週のどこか**を常に意識できるプログレスバー
- 残りの日数と未完了タスクの比率でプレッシャー感
- 週末（日曜）までにDone = 1週間サイクル成功

#### B. 担当者ビュー
```
サノ        中          ししょー
┌────┐    ┌────┐      ┌────┐
│ 2件 │    │ 3件 │      │ 1件 │
│ 進行 │    │ 進行 │      │ 待ち │
└────┘    └────┘      └────┘
```
- 誰が何を持っているかワンタップで確認
- 偏りがあれば再分配の判断材料

#### C. タスクカード
- 担当者をアバターで表示（サ/中/師の色付き丸）
- 完了チェックは**スワイプ右**で
- 成果物URL（GitHub Pages等）があればリンクプレビュー

#### D. プロダクト一覧・成果物管理（★追加）
```
┌──────────────────────────────────────┐
│ 📦 プロダクト一覧            [+ 登録] │
├──────────────────────────────────────┤
│ ┌────────────────────────────────┐   │
│ │ 🟢 Live  モルック団体管理アプリ    │   │
│ │ Lead: 中  |  完成: 2026/03/07  │   │
│ │ URL: jonyjean21.github.io/...  │   │
│ │ 技術: HTML/JS/Firebase         │   │
│ │ カテゴリ: 🎯モルック              │   │
│ └────────────────────────────────┘   │
│ ┌────────────────────────────────┐   │
│ │ 🟡 Dev   コミュニティマップ       │   │
│ │ Lead: サノ  |  進捗: 70%       │   │
│ │ Sprint: 3/3〜3/9               │   │
│ └────────────────────────────────┘   │
│ ┌────────────────────────────────┐   │
│ │ 📋 Plan  イベント告知ボット      │   │
│ │ Lead: ししょー  |  FIND段階      │   │
│ └────────────────────────────────┘   │
├──────────────────────────────────────┤
│ 合計: 8件  Live: 3 / Dev: 2 / Plan: 3 │
└──────────────────────────────────────┘
```
- **全プロダクトの一覧管理**: ステータス（Plan/Dev/Live/Archive）
- FINDで採用→MAKEでProject化したものが自動で登録
- 成果物URL、担当者、技術スタック、完成日を記録
- カテゴリ（FINDと共通）で絞り込み可能
- OFFER/PRESENTとの紐付け（このプロダクトを誰に提供したか、どこで発信したか）

## 3. OFFER — 提供管理

### 現状の問題
- 単なるリスト。提供フローの可視化がない
- 現段階では具体的な「提供」よりも「オープンにする」がスタート

### 改修案（現段階の規模感に合わせたシンプル版）

#### A. 提供パイプライン
```
┌─────────┐  ┌─────────┐  ┌─────────┐
│ 📤 公開済 │  │ 💬 FB中  │  │ ✅ 完了  │
│   3件    │  │   1件    │  │   2件    │
│         │  │         │  │         │
│ A (公開)  │  │ B (FB中) │  │ C (完了) │
│ D (公開)  │  │         │  │ E (完了) │
│ F (公開)  │  │         │  │         │
└─────────┘  └─────────┘  └─────────┘
```
- まずは「公開した（オープンにした）」をスタート地点として記録
- 具体的な提供先が出てきたら提供先管理に発展

#### B. フィードバック収集
- 提供先ごとに「反応メモ」を時系列で追記
- 絵文字リアクション（😊👍🤔😞）でざっくり評価
- フィードバックから次のFINDネタに自動リンク

#### C. 提供実績サマリー
- 累計公開数、フィードバック率
- プロダクト別の提供・公開状況一覧

## 4. PRESENT — 発信管理

### 現状の問題
- 投稿記録としては機能するが、戦略性がない
- **チャネルを意識しない人間が自然に発信するのは現実的ではない**
- 「発信しなきゃ」と思っても何をどこに発信すればいいかわからず止まる

### 改修案 — 仕組みで発信を回す

#### A. 発信プラン（チャネル×コンテンツのマトリクス）
```
┌──────────────────────────────────────────┐
│ 📢 発信プラン                    [+ 追加] │
├──────────────────────────────────────────┤
│ プロダクト: モルック団体管理アプリ          │
│                                          │
│ チャネル       状態      予定/実績         │
│ ┌────────────────────────────────────┐   │
│ │ 𝕏 X          [未着手]   →テンプレあり │   │
│ │ 📸 Instagram  [下書き]   3/10予定     │   │
│ │ 🎤 イベント    [完了]    3/8 LT発表   │   │
│ │ 🗣 口コミ      [完了]    中さん経由    │   │
│ └────────────────────────────────────┘   │
│                                          │
│ 発信率: 2/4チャネル完了 (50%)             │
└──────────────────────────────────────────┘
```
- **プロダクトごとに「どのチャネルで発信するか」を計画**
- チャネル: X / Instagram / イベント(LT等) / 口コミ / ブログ / その他
- 各チャネルの状態: 未着手 / 下書き / 投稿済み / 完了
- **発信率**を可視化 → 「Xだけで終わってない？Instagramもやろう」

#### B. 発信テンプレート＆下書きサポート
```
┌──────────────────────────────────┐
│ 📝 テンプレートから下書き作成       │
│                                  │
│ [X用: 短文+URL]                  │
│ [Instagram用: ビジュアル+説明]     │
│ [イベント用: LTスライド概要]       │
│                                  │
│ → テンプレ選択で下書きが自動生成   │
│   （Gemini活用で内容に合わせて）    │
└──────────────────────────────────┘
```
- プロダクト情報から**チャネル別の下書きを自動生成**（Gemini活用）
- X用は短文+ハッシュタグ、Instagram用はビジュアル重視の説明文
- 生成された下書きを確認→コピーして投稿
- よく使うフォーマットをテンプレートとして保存

#### C. 発信カレンダー
```
    3月
月 火 水 木 金 土 日
                1  2
 3  4  5  6  7  8  9
[X]    [IG]      [X]
10 11 12 13 14 15 16
   [Ev]         [X]
```
- チャネル別アイコンで月間の発信頻度を可視化
- 空白日 = 発信していない日 → 頻度改善の意識づけ

#### D. チャネル別サマリー
```
┌──────────────────────────┐
│ チャネル別 発信状況        │
│                          │
│ 𝕏 X:        12件 (主力)   │
│ 📸 Instagram:  3件 (弱い)  │
│ 🎤 イベント:    2件        │
│ 🗣 口コミ:      5件        │
│                          │
│ → Instagramが手薄。       │
│   テンプレ使って投稿しよう  │
└──────────────────────────┘
```
- どのチャネルが効果的か・手薄かを可視化
- 弱いチャネルに対してアクション提案

## 横断機能

### プロジェクト詳細ビュー
各プロジェクトをタップすると、5STEPそれぞれの進捗を1画面で確認：
```
┌──────────────────────┐
│ PJ: モルック団体管理    │
│ Lead: 中  Status: Active │
│                      │
│ FIND   ████████░░ 80% │
│ MAKE   ██████░░░░ 60% │
│ OFFER  ██░░░░░░░░ 20% │
│ PRESENT ░░░░░░░░░░ 0% │
│ CONSULT ░░░░░░░░░░ 0% │
│                      │
│ タイムライン:          │
│ 3/1 アイデア登録       │
│ 3/3 チャプ会で採用決定  │
│ 3/4 開発開始（中担当）  │
│ 3/7 v1完成・提供開始    │
└──────────────────────┘
```

### 自動分類の仕組み（Gemini活用）
- クイック投稿時にGemini APIで内容を解析
- 判定項目:
  - **カテゴリ**: モルック / コミュニティ / ビジネス / ツール / その他
  - **STEP**: FIND / MAKE / OFFER / PRESENT / CONSULT
- 判定結果はサジェスト（ユーザーが変更可能）
- 学習データ: 過去の投稿と分類結果をもとに精度向上（将来）

### スマホ対応の設計方針
- **ボトムナビ**: 5STEP + Dashboard の6タブ（スマホ時）
- **スワイプジェスチャー**: カード操作（ステータス変更）
- **フローティングアクションボタン**: 右下に「+」で即投稿（自動分類付き）
- **プルトゥリフレッシュ**: 画面引っ張って更新
- **タップ→展開**: カードタップで詳細展開（モーダルではなくインライン展開）
- **最小タップ数**: 投稿は2タップ以内（FAB→入力→送信→自動分類）
