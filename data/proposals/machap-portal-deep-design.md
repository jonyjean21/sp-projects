# MACHAP Portal — 各STEP機能深掘り計画書

## 基本方針
- **スマホファースト**: チーム3人がスマホで使う前提。PCはサブ
- **入力のハードルを極限まで下げる**: 思いついた瞬間に投稿できる簡単さ
- **可視化で意思決定を助ける**: 次に何を作るか、3人で選べる画面
- **5STEPが回っている実感**: サイクルの進行を視覚的に体感できる

---

## 1. FIND — 課題・アイデア発見ボード

### 現状の問題
- 単なるリスト表示で、投稿しやすさ・選びやすさが弱い
- アイデアの質を評価する仕組みが投票ボタンだけ

### 改修案

#### A. クイック投稿（スマホ最適化）
```
┌──────────────────────────────┐
│ 💡 アイデアを1行で             │
│ [____________________________] │
│                     [投稿 →]  │
└──────────────────────────────┘
```
- **1行入力で即投稿**。タイトルだけでOK（詳細は後から編集）
- 投稿後にカード展開して概要・ターゲットを追記可能
- LINEに送る感覚で使える軽さ

#### B. カンバンビュー（横スクロール）
```
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ 💡 発見  │  │ 🔍 検討  │  │ ✅ 採用  │  │ 🚀 実行  │
│         │  │         │  │         │  │         │
│ カードA  │  │ カードC  │  │ カードE  │  │         │
│ カードB  │  │ カードD  │  │         │  │         │
│         │  │         │  │         │  │         │
└─────────┘  └─────────┘  └─────────┘  └─────────┘
        ←  スワイプで移動  →
```
- スマホではカードを**長押し→ドラッグ**でステータス変更
- または**スワイプジェスチャー**で右に送る = ステータス進行
- 各列のカード数がひと目でわかる

#### C. 評価マトリクス（選びやすさ）
```
           影響度 大
              ↑
    ┌─────────┼─────────┐
    │ 検討枠  │ 即やる！  │
    │         │  ★カードE │
    │─────────┼─────────│
    │ 保留    │ 余裕あれば │
    │         │         │
    └─────────┼─────────┘
              →
           実現容易度 高
```
- アイデアに「影響度」「実現容易度」の2軸スコア（1-5）を設定
- マトリクスに自動プロット
- **右上（高影響×容易）= 次に作るべきもの**が一目瞭然
- 3人で議論しやすい可視化

#### D. アイデアカード詳細
```
┌──────────────────────────┐
│ 💡 モルック団体管理アプリ    │
│ ────────────────────────  │
│ 概要: 各団体の練習日・     │
│   メンバー管理を一元化     │
│                          │
│ ターゲット: モルック団体代表  │
│ 課題: 練習参加連絡がバラバラ │
│                          │
│ 影響度: ★★★★☆  容易度: ★★★☆☆ │
│                          │
│ [サ✓] [中✓] [師○]  投票: 2  │
│                          │
│ ステータス: [検討中 ▼]      │
│ 関連メモ: チャプ会260303で話題 │
│                          │
│ [編集] [Project化→] [削除]  │
└──────────────────────────┘
```

## 2. MAKE — プロダクト化トラッカー

### 現状の問題
- タスクリストは普通のTODOと変わらない
- 「1週間でプロダクト化」のスピード感が可視化されていない

### 改修案

#### A. スプリントボード（週次ビュー）
```
┌──────────────────────────────────┐
│ Sprint: 3/3(月) 〜 3/9(日)        │
│ ━━━━━━━━━━━━━━━━━━━━ 60%        │
│                                  │
│ 月 火 水 木 金 土 日              │
│ ── ── ── ██ ░░ ░░ ░░   ← 今日   │
│                                  │
│ [TODO: 3]  [進行中: 2]  [完了: 4] │
└──────────────────────────────────┘
```
- **今日が週のどこか**を常に意識できるプログレスバー
- 残りの日数と未完了タスクの比率でプレッシャー感
- 週末（日曜）までにDone = 1週間サイクル成功

#### B. 担当者ビュー
```
サノ        中          ししょー
┌────┐    ┌────┐      ┌────┐
│ 2件 │    │ 3件 │      │ 1件 │
│ 進行 │    │ 進行 │      │ 待ち │
└────┘    └────┘      └────┘
```
- 誰が何を持っているかワンタップで確認
- 偏りがあれば再分配の判断材料

#### C. タスクカード
- 担当者をアバターで表示（サ/中/師の色付き丸）
- 完了チェックは**スワイプ右**で
- 成果物URL（GitHub Pages等）があればリンクプレビュー

## 3. OFFER — 提供管理

### 現状の問題
- 単なるリスト。提供フローの可視化がない

### 改修案

#### A. 提供パイプライン
```
┌─────────┐  ┌─────────┐  ┌─────────┐
│ 📤 提供済 │  │ 💬 FB中  │  │ ✅ 完了  │
│   3件    │  │   1件    │  │   2件    │
│         │  │         │  │         │
│ A社      │  │ B団体    │  │ C個人    │
│ D団体    │  │         │  │ E社      │
│ F個人    │  │         │  │         │
└─────────┘  └─────────┘  └─────────┘
```

#### B. フィードバック収集
- 提供先ごとに「反応メモ」を時系列で追記
- 絵文字リアクション（😊👍🤔😞）でざっくり評価
- フィードバックから次のFINDネタに自動リンク

#### C. 提供実績サマリー
- 累計提供先数、フィードバック率、満足度
- プロダクト別の提供状況一覧

## 4. PRESENT — 発信管理

### 現状の問題
- 投稿記録としては機能するが、戦略性がない

### 改修案

#### A. 発信カレンダー
```
    3月
月 火 水 木 金 土 日
                1  2
 3  4  5  6  7  8  9
[X]    [IG]      [X]
10 11 12 13 14 15 16
   [Ev]         [X]
```
- チャネル別アイコンで月間の発信頻度を可視化
- 空白日 = 発信していない日 → 頻度改善の意識づけ

#### B. チャネル別ダッシュボード
- X / Instagram / イベント / 口コミ それぞれの件数・リーチ
- どのチャネルが効果的か比較

#### C. テンプレート（下書き保存）
- よく使う発信フォーマットを保存
- ボタン一発で下書き作成

## 5. CONSULT — 相談案件管理

### 現状の問題
- 案件リストとして機能するが、ビジネスとしての追跡が弱い

### 改修案

#### A. 案件パイプライン
```
受付 → ヒアリング → 提案 → 実行中 → 完了
 2       1          0      1       3
```
- 営業パイプライン的な管理
- 各フェーズの件数を可視化

#### B. 収益ダッシュボード
```
┌──────────────────────┐
│ 今月: ¥50,000        │
│ 累計: ¥150,000       │
│                      │
│ 配分:                │
│  チーム 30%: ¥45,000  │
│  個人  30%: ¥45,000  │
│  外部  40%: ¥60,000  │
└──────────────────────┘
```
- 収益モデル（30/30/40）に基づく自動配分計算
- 月別推移グラフ

#### C. 相談→プロダクト化の流れ
- 相談内容から新しいFINDアイデアを生成するボタン
- 5STEPサイクルが相談からも回る仕組み

## 横断機能

### プロジェクト詳細ビュー
各プロジェクトをタップすると、5STEPそれぞれの進捗を1画面で確認：
```
┌──────────────────────┐
│ PJ: モルック団体管理    │
│ Lead: 中  Status: Active │
│                      │
│ FIND   ████████░░ 80% │
│ MAKE   ██████░░░░ 60% │
│ OFFER  ██░░░░░░░░ 20% │
│ PRESENT ░░░░░░░░░░ 0% │
│ CONSULT ░░░░░░░░░░ 0% │
│                      │
│ タイムライン:          │
│ 3/1 アイデア登録       │
│ 3/3 チャプ会で採用決定  │
│ 3/4 開発開始（中担当）  │
│ 3/7 v1完成・提供開始    │
└──────────────────────┘
```

### スマホ対応の設計方針
- **ボトムナビ**: 5STEP + Dashboard の6タブ（スマホ時）
- **スワイプジェスチャー**: カード操作（ステータス変更）
- **フローティングアクションボタン**: 右下に「+」で即投稿
- **プルトゥリフレッシュ**: 画面引っ張って更新
- **タップ→展開**: カードタップで詳細展開（モーダルではなくインライン展開）
- **最小タップ数**: 投稿は2タップ以内（FAB→入力→送信）
