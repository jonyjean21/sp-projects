# 引用リポスト生成スキル

## 概要
モルハブ（@molkkyhub）宛の言及ポストに対して、引用リポストの文面を自動生成する。
モルハブの人格設定（molkkyhub-persona.md）に基づいたトーンで統一。

## 使い方
- `/repost` — 直前のメッセージ（テキストまたはスクショ）から引用リポスト案を生成
- `/repost <テキスト>` — 指定テキストに対する引用リポスト案を生成

## 処理フロー

### Step 1: 元ポストの内容を把握
ユーザーから以下のいずれかで元ポストを受け取る:
- スクリーンショット（画像読み取り）
- テキストコピペ
- ポストURL（syndication APIで取得を試みる）

抽出する情報:
- 投稿者名 / @ユーザー名
- ポスト本文
- 文脈（何について言及しているか）
- モルハブとの関係性（協賛、寄稿、言及、感謝など）

### Step 2: 過去のモルハブポストを参照（任意）
X syndication APIでモルハブの直近ポストを取得し、トーンの一貫性を確認:

```bash
curl -sf "https://syndication.twitter.com/srv/timeline-profile/screen-name/molkkyhub" \
  -H "User-Agent: Mozilla/5.0" | grep -o '"text":"[^"]*"' | head -15
```

### Step 3: 人格設定を読み込み
`/Users/shumpei/sp-projects/.claude/skills/molkkyhub-persona.md` を参照し、
トーン・スタイル・原則に従って文面を生成する。

### Step 4: 引用リポスト案を2〜3パターン生成

#### 生成ルール
1. **人格設定に厳密に従う**（molkkyhub-persona.md 参照）
2. **相手を主役にする**: モルハブは黒子。相手の活動を称える
3. **補足価値を添える**: ハブとして情報を繋げる一言
4. **文字数**: 100〜200文字程度（引用リポストなので短めでOK）
5. **標準語**: 関西弁は使わない
6. **絵文字**: 2〜4個程度
7. **ハッシュタグ**: 末尾に `#モルック #molkky`
8. **自己主張控えめ**: 「モルハブすごい」的な自画自賛は絶対にしない

#### パターンの差
- **A. 感謝・称賛型**: 相手への感謝を軸に
- **B. 情報補足型**: 関連情報やモルハブでの掲載予定を添える
- **C. つながり型**: モルック界の広がりに触れる

### Step 5: ユーザーに提示

AskUserQuestion で提示:

```
📱 引用リポスト案

元ポスト: @{ユーザー名} の {内容要約}

【A】感謝・称賛型
{パターンA}

【B】情報補足型
{パターンB}

【C】つながり型
{パターンC}

どれを使いますか？（修正も可能です）
```

### Step 6: 最終出力
選択されたポスト文面をコピペ用に出力:

```
━━━ 引用リポスト（コピペ用） ━━━

{選択されたポスト文面}

━━━━━━━━━━━━━━━━━

文字数: XXX文字
```

## 注意事項
- 常に molkkyhub-persona.md の人格設定を遵守
- 自画自賛・宣伝臭は排除
- 元ポストの文脈を正確に理解してから生成
- 相手の名前・アカウント名を正確に記載
- スクショの場合は画像を読み取って内容を把握
