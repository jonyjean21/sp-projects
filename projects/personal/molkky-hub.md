# MOLKKY HUB

> モルックを愛するすべてのひとのために

## 概要

モルックの総合情報メディアサイト。ルール解説・大会情報・チーム紹介・ツール紹介など、日本のモルック情報を一元的に集約する「ハブ」を目指して運営中。

## 基本情報

| 項目 | 内容 |
|------|------|
| URL | https://molkky-hub.com |
| ステータス | 運用中 |
| 運営 | 個人（管理人：孔雀） |
| 技術スタック | WordPress + SWELL テーマ |

## サイト構成

- **ルール・遊び方** — 初心者向けの基本ルール解説、ゲームの流れ・得点計算・勝敗の決まり方
- **大会情報** — 全国のモルック大会・練習会の情報掲載
- **チーム紹介** — 全国のモルックチームのプロフィールページ
- **記事一覧** — モルックの人気・魅力に関する記事コンテンツ
- **ツール** — スコア管理アプリまとめ等
- **正規品ガイド** — TACTIC社製モルック全ラインナップの紹介
- **YouTubeチャンネル一覧** — モルック関連YouTuberのディレクトリ
- **NICE MOLKKY JAPAN** — 自主大会の運営（大会詳細・レギュレーション・ギャラリー等）

## 今後の取り組み

### 1. 大会情報掲載の自動化

**ゴール:** 大会・練習会情報の収集〜掲載を自動化し、運用負荷を大幅に減らす

**パイプライン構成:**
```
X タイムライン → Google スプレッドシート → GAS → WordPress 投稿
```

**アクションプラン:**

| # | タスク | 詳細 |
|---|--------|------|
| 1 | 情報ソースの整理 | 大会情報を発信しているXアカウント・ハッシュタグをリストアップ |
| 2 | スプレッドシート設計 | 大会名・日時・場所・主催者・申込URL・ステータス等のカラム設計 |
| 3 | X → スプシの収集方法検討 | X API（有料）/ IFTTT / 手動＋半自動のどれで行くか判断 |
| 4 | WordPress REST API の有効化 | Application Passwords または JWT 認証でAPI経由の投稿を可能にする |
| 5 | GAS スクリプト開発 | スプシの新規行を検知 → WordPress REST API で投稿を自動作成 |
| 6 | 投稿テンプレート作成 | SWELL のカスタムブロック構成に合わせたHTML/投稿フォーマット設計 |
| 7 | テスト＆運用開始 | 下書き投稿で動作確認 → トリガー設定（定期実行）→ 本番運用 |

**技術選定のポイント:**
- X API は Free プランだと制約が大きい。まずは手動収集 + スプシ管理 → GAS で WordPress 投稿の流れから始めるのが現実的
- GAS のトリガーで定期実行（例: 1日1回）するだけで十分な頻度

---

### 2. 記事投稿の半自動化

**ゴール:** 記事作成〜投稿の工数を減らし、コンテンツ量を拡大する

**アクションプラン:**

| # | タスク | 詳細 |
|---|--------|------|
| 1 | 記事カテゴリの整理 | 定型記事（チーム紹介・大会レポ等）と非定型記事（コラム・解説等）を分類 |
| 2 | 定型記事テンプレート作成 | チーム紹介・大会レポなど、構成が決まっている記事のテンプレートをスプシ or Notion で用意 |
| 3 | AI 活用の検討 | テンプレート + 入力情報 → Claude 等で下書き生成 → 人間がレビュー・編集 → 投稿 |
| 4 | WordPress 投稿フロー整備 | GAS or WP REST API でスプシ/Notion → WordPress への投稿を半自動化 |
| 5 | 運用ルール策定 | レビューフロー・公開判断基準・更新頻度の目安を決める |

**ポイント:**
- 完全自動ではなく「半自動」がカギ。品質担保のため人間のレビューは残す
- チーム紹介のような定型コンテンツから始めると効果を実感しやすい

---

### 3. スポンサー獲得に向けた施策

**ゴール:** サイト運営の持続可能性を確保し、活動を拡大する

**アクションプラン:**

| # | タスク | 詳細 |
|---|--------|------|
| 1 | 実績・数値の整理 | PV数・UU数・SNSフォロワー数・大会開催実績などをまとめる |
| 2 | メディアキット作成 | サイト概要・読者層・リーチ数・広告メニュー（バナー・記事広告・大会協賛等）をPDF化 |
| 3 | ターゲットスポンサーのリストアップ | モルック用品メーカー（TACTIC等）・スポーツ用品店・地方自治体・アウトドアブランド等 |
| 4 | 収益モデルの設計 | バナー広告 / 記事広告 / 大会スポンサー / アフィリエイト の組み合わせを検討 |
| 5 | 営業開始 | メディアキットを持って打診。まずはモルック関連企業・団体から |

**ポイント:**
- まずはアフィリエイト（Amazon等でモルック用品紹介）で小さく始めるのもあり
- 大会運営（NICE MOLKKY JAPAN）の実績は大きな武器になる

## 優先度の考え方

```
高  ┌─────────────────────────────┐
影  │  ① 大会情報自動化           │ ← 運用負荷削減 × すぐ着手可能
響  │  ② 記事半自動化             │ ← コンテンツ拡大 × ①の基盤を活用
度  │  ③ スポンサー獲得           │ ← ①②でサイト価値を上げてから
低  └─────────────────────────────┘
    すぐできる ───────────── 時間がかかる
```

①→②→③ の順で着手するのが自然な流れ。①②で自動化基盤を整えてサイトの更新頻度と価値を上げた上で③に臨むのが効果的。
