## 前提: Claude Codeとは何か

Claude Codeはターミナル（コマンドライン）で動くAIアシスタントだ。ブラウザ版のClaude（claude.ai）とは別のプロダクト。

ブラウザ版との最大の違いは**ファイル操作とコマンド実行ができること**。AIがファイルを読み書きし、gitコマンドを打ち、curlでAPIを叩き、コードを実行する。チャットで会話するだけのAIとは次元が違う作業ができる。

俺はこれを使って以下の作業を全自動化している。

- WordPressへの記事自動投稿（550件以上）
- アフィリエイト記事の自動生成
- 大会情報の自動収集と整形
- 議事録の自動変換（音声文字起こし → 構造化ドキュメント）
- クラウドワークス提案文の生成
- note記事の下書き生成
- Xポスト案の生成

全部AIがやっている。俺がやるのは最終確認とボタンを押すことだけだ。

## 「無料枠でできること」の限界

「まずは無料で試したい」という人は多い。Anthropicには無料のAPIクレジットがある（新規登録時に$5）。これで何ができるかを計算する。

$5（約¥750）でHaikuを使った場合:
- 入力: 約3,000,000トークン（約150万文字の入力）
- 出力: 約600,000トークン（約30万文字の出力）

つまり**2,000文字の記事を約150本生成できる**。

$5の無料枠だけでアフィリエイト記事150本。これだけでサイトの骨格は作れる。

### 無料 vs 有料で何が違うか

| 項目 | 無料枠（$5） | 有料投資（月¥5,000） |
|------|-------------|---------------------|
| 記事生成 | 150本（使い切り） | 月300本以上 |
| モデル | Haiku限定が現実的 | Haiku + Sonnet + Opus使い分け |
| コード生成 | Haikuだと品質に限界 | Opusで本格的な開発可能 |
| 議事録・長文処理 | コンテキスト不足 | Sonnetで高品質処理 |
| 自動化パイプライン | 構築まではできる | 運用し続けられる |

**結論: 無料枠でまず試す → 手応えを感じたら月¥3,000〜5,000に投資する。**

いきなり月¥10,000とか投資する必要はない。Haikuメインで月¥1,000〜2,000から始めて、収益が出たら少しずつ投資額を上げればいい。

## APIに投資して回収できるのか？：正直な回答

ここが一番聞かれる質問だ。正直に答える。

**「回収できるかどうかは、仕組みを作れるかどうかで決まる」。**

AIにチャットで質問するだけでは回収できない。ブラウザ版Claude Pro（月$20）で「ブログ記事書いて」とお願いしても、コピペしてWordPressに貼り付けるのは自分。投稿するのも自分。SNSに告知するのも自分。手作業がボトルネックになる。

回収できるのは「AIを使って仕組みを作った場合」だ。記事を自動生成して、自動投稿して、自動で告知する。この仕組みがあって初めて、AIの費用を収益で回収するサイクルが回る。

もう一つ正直に言うと、**最初の1〜2ヶ月は赤字になる覚悟が必要**。仕組みの構築期間はAPI費用を使いながら収益はゼロだ。俺も1ヶ月目は完全赤字だった。

ただし、仕組みが完成した瞬間から費用は固定になり、収益は右肩上がりになる。これが従量課金の良さでもある。使わなければ¥0、使った分だけ払う。月額固定のSaaSと違って、閑散期にコストが膨れない。

## 費用を抑える5つのコツ

1. **Haikuファーストの原則** — まずHaikuで試す。品質が足りない時だけ上位モデルに上げる
2. **プロンプトを短くする** — 入力トークンもお金。冗長な指示は無駄。テンプレート化して再利用する
3. **バッチ処理でまとめて実行** — 1件ずつ処理するとオーバーヘッドが大きい。50件まとめて処理する
4. **キャッシュを使う** — 同じ情報の取得は1回だけ。結果をファイルに保存して再利用する
5. **OpusはMCPツールの開発とアーキテクチャ設計だけ** — 日常の文章生成にOpusは贅沢すぎる

この5つを守るだけで、同じ作業量でAPI費用が半分以下になる。

この事実を知っているだけで、AIへの投資に対する見方が変わるはずだ。問題は「いくらかかるか」ではなく「何を作るか」だ。
