## はじめに

### Claude Code / Claude APIとは

Claude Codeは、Anthropic社が提供するAIアシスタント「Claude」をコマンドラインから直接操作できる公式CLIツールです。ターミナル上でファイル操作、コード生成、テキスト作成を一括で行えるため、従来のチャットUIでは実現できなかった「作業の自動化」が可能になります。

Claude APIは、同じくAnthropicが提供するAPI経由でClaudeの能力をプログラムから呼び出す仕組みです。GAS（Google Apps Script）やNode.js、Pythonなどから直接呼び出して、定期実行や自動処理パイプラインに組み込めます。

### この本の使い方

本書に掲載されている50個のプロンプトは、全て**そのままコピペして使える実践テンプレート**です。

1. **カテゴリから選ぶ** — 自分の副業スタイルに合ったカテゴリを選択
2. **プロンプトをコピー** — `[変数]` の部分を自分の情報に置き換える
3. **Claude Code / APIに入力** — 出力をそのまま業務に活用
4. **使い方のコツを参照** — より良い結果を得るための調整方法を確認

各プロンプトの `[角括弧]` 内は、あなた自身の情報に置き換えてください。例えば `[ターゲットキーワード]` なら「モルック 始め方」のように具体的な語句を入れます。

**重要:** プロンプトの出力はあくまで「たたき台」です。最終的には自分の目で確認し、事実確認・修正を行ってから公開してください。AIの出力をそのまま使うのではなく、自分の知見や体験を加えることで、競合と差別化できる質の高いコンテンツになります。

### プロンプト01: 検索上位を狙うSEO記事の一発生成

**プロンプト本文:**

```
あなたはSEOライティングの専門家です。以下の条件でGoogle検索上位を狙える記事を作成してください。

【ターゲットキーワード】[メインキーワード]
【検索意図】[情報収集 / 比較検討 / 購入検討 のいずれか]
【想定読者】[読者のペルソナ: 年齢層、知識レベル、悩み]
【記事の文字数】3,000〜5,000文字
【競合との差別化ポイント】[自分だけが書ける体験や知見]

以下の構成で記事を生成してください:
1. タイトル（32文字以内、キーワードを前半に配置）
2. メタディスクリプション（120文字以内）
3. リード文（読者の悩みに共感→この記事で解決できると提示）
4. H2見出し×4〜6個（キーワードの関連語を含む）
5. 各H2の下にH3を2〜3個
6. 本文（具体的なデータ・事例・手順を含む）
7. まとめ（要点3つ + 読者への行動提案）

E-E-A-T（経験・専門性・権威性・信頼性）を意識し、特に「経験」に基づく一次情報を盛り込んでください。
```

**使い方のコツ:**
- `[競合との差別化ポイント]` が最も重要。自分の実体験や独自データを必ず入れること。
- 出力後、Googleで同じキーワードを検索して上位10記事と被っていないか確認する。
- 一度で完璧な記事は出ないので、「H2の3番目をもっと具体的に」など追加指示で磨く。

**期待される出力例:**
タイトル「モルックの始め方ガイド|初心者が最初に知るべき5つのこと」に続き、見出し構成・メタディスクリプション・本文が一括で生成される。

### プロンプト04: FAQ構造化記事の自動生成

**プロンプト本文:**

```
以下のテーマについて、Google検索の「他の人はこちらも質問」欄に表示されそうなFAQ（よくある質問）を推定し、FAQ構造化データ（JSON-LD）付きの記事を生成してください。

【テーマ】[テーマ名]
【想定読者の知識レベル】[初心者 / 中級者 / 上級者]
【カバーしたい範囲】[具体的な範囲の指定]

1. FAQ 15〜20個（質問と回答のペア、各回答は100〜200文字）
2. FAQを論理的にグループ分けした見出し構成（H2×4〜5個）
3. 各グループのリード文（読者を次のFAQに誘導する文）
4. JSON-LD構造化データ（FAQPage schema）
5. メタディスクリプション

回答には「〜です。」で終わる断定形を使い、具体的な数値や手順を含めてください。曖昧な表現（「場合もあります」「人によります」）は最小限に。

**使い方のコツ:**

- FAQ記事は検索結果でリッチスニペット表示される可能性があり、CTRが大幅に向上する。

**期待される出力例:**
Q「モルックは何人で遊べますか？」A「モルックは2人から16人程度で遊べます。公式ルールでは1チーム1〜6人で...」という形式のFAQ20個とJSON-LDコード。

### プロンプト10: ハッシュタグ最適化

**プロンプト本文:**

```
以下の投稿内容に最適なハッシュタグの組み合わせを提案してください。

【投稿内容】
「[投稿の本文]」

【アカウントのジャンル】[ジャンル]
【目的】[新規リーチ拡大 / コミュニティ内での露出 / トレンド便乗]
【フォロワー規模】[〜500 / 500〜2000 / 2000〜10000 / 10000以上]

以下の3セットを提案してください:
1. 安全セット（定番タグ3〜5個、安定したリーチが見込める）
2. 攻めセット（トレンドタグ＋ニッチタグの組み合わせ5〜7個）
3. 最小セット（最も効果的な2個だけ）

各ハッシュタグについて:
- 推定利用頻度（高 / 中 / 低）
- 選定理由（1行）
- 注意点（あれば）

また、以下も併せて回答:
- このジャンルで「使うべきでないNGタグ」とその理由
- ハッシュタグを入れる最適な位置（本文中 / 末尾 / リプ欄）
```

**使い方のコツ:**
- Xではハッシュタグの効果はInstagramほど高くない。2〜3個が最適で、多すぎるとスパム扱いされる。
- 「最小セット」を基本とし、特定のイベントやトレンドがある時だけ「攻めセット」を使う。
- 定期的にタグの効果をXアナリティクスで確認し、効果のないタグは外す。

**期待される出力例:**
安全セット「#モルック #アウトドアスポーツ #週末の遊び」/ 攻めセット「#モルック #ニュースポーツ #子供と遊ぶ #公園遊び #休日の過ごし方」など。
