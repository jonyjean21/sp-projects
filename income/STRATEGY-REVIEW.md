## 3. ¥155,000 は現実的か？

**正直な回答: 3月中にはほぼ不可能。**

理由:
1. **全てのプラットフォームがゼロスタート**。BOOTH、note、CW、Xの全てでフォロワー/実績/評価がゼロ。これが最大のボトルネック
2. **デジタル商品は「信頼」で売れる**。無名の人間が ¥1,980 の商品を20個売るには、まず信頼を構築する期間が必要
3. **受託は評価経済**。CWで実績ゼロから ¥50,000 案件を取るのは確率が低い
4. **アフィリエイトは時間がかかる**。SEO記事が検索に反映されるまで2〜3ヶ月

**¥155,000 に到達するとしたら、それは5月〜6月になるだろう。**

3月は「¥30,000を稼ぎつつ、全チャネルの種まきを完了させる月」と位置づけるのが正しい。

## 6. 最大のリスクと対策

### リスク1: 時間の分散（最大のリスク）

**問題**: 6つのプラットフォーム（BOOTH, note, CW, X, ココナラ, アフィリエイト）を同時に立ち上げようとすると、全てが中途半端になる

**対策**:
- 第1週は「ココナラ出品 + MOLKKY HUBアフィリエイト最適化 + AdSense申請」の3つだけに集中
- 第2週以降に CW、note、BOOTH を順次追加
- SPの作業時間は1日15分が限界。やることを絞る

### リスク2: 「稼いでいないのに稼いでいるように書く」問題

**問題**: note記事の article-01 は「Claude Codeで副業を始めて3ヶ月。実際の収益を全部公開する」というタイトルだが、内容は架空の実績ストーリー。「2ヶ月目は3,000円、3ヶ月目は10,000円」と書いてあるが、実際にはまだその実績がない

**対策**:
- 嘘の実績を書くのは長期的に信頼を毀損する。バレたらブランド崩壊
- 「これから挑戦する」「Day 1 の記録」として正直に書く方が、かえって共感を得られる
- 「収益ゼロからスタート。リアルタイムで公開していく」というスタンスに変更すべき

### リスク3: 受託案件の納品リスク

**問題**: Claude Codeで開発できるとはいえ、クライアントとのコミュニケーション、仕様変更対応、バグ修正は人間の作業。SPの時間が限られている中で受託を受けると、納期遅延のリスク

**対策**:
- 最初は小規模案件（¥10,000以下）に限定
- 納期に余裕を持つ（提示納期の1.5倍を見積もる）
- 「追加要件は別料金」を最初に明示

### リスク4: プラットフォーム手数料の見落とし

**問題**:
- BOOTH: 手数料 5.6% + 22円/件（2025年10月改定後は45円）→ ¥1,980の商品で手取り約¥1,847
- ココナラ: 22% → ¥10,000で手取り¥7,800
- note: プラットフォーム利用料10% + 決済手数料5% = 15% → ¥1,980で手取り約¥1,683
- CW: システム利用料 20%（10万円以下の部分）→ ¥50,000で手取り¥40,000

**対策**: 収益計画には手数料を織り込んで計算する

### リスク5: 全自動化への過度な期待

**問題**: 「全部Claudeが自動でやる」という前提は、初期セットアップと各プラットフォームのアカウント作成が手動であることを見落としている。cron で記事生成を自動化しても、noteへの投稿、BOOTHへの出品、CWへの応募は全て手動操作が必要

**対策**:
- Claude Code ができるのは「コンテンツ生成」と「コード作成」まで
- プラットフォームへの投稿・出品は SPが手動でやる必要がある
- 1日15分のルーティンを守り、その中で処理できる量に絞る
