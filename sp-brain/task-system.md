# タスク管理システム

## コンセプト
SPは **Todoist にメモを入れるだけ**。それ以外は何もしなくていい。
分類も、タスク化も、リマインドも、完了処理も、全部AI秘書がやる。

---

## メモの入口: Todoist のみ

### なぜ Todoist か
- スマホウィジェットで3秒入力
- Ramble（音声）で話すだけ
- 自然言語で期限・プロジェクト自動認識
- Claude Code と MCP で直結（Doist公式）
- 無料で十分

### 入力ルール（SPが覚えること）
```
なし。好きなように入れろ。
```

例:
- 「来月ブースやりたい」 → Inbox に入る
- 「中さんに日程確認」 → Inbox に入る
- 「明日 記事構成 #MOLKKYHUB」 → 自動で振り分けされる
- 音声で「パートナーの返信確認せなあかん」 → Ramble が文字化して Inbox へ

**プレフィックス不要。キーワード不要。ただ思ったことを入れる。**

---

## AI秘書の処理フロー

```
Todoist Inbox に何か入る
  ↓
セッション開始時に MCP で取得
  ↓
AI秘書が1件ずつ判定:

  「来月ブースやりたい」
    → タスクやな → Todoist #MOLKKYHUB に移動、期限「来月」
    → tasks/current.md にも追加

  「中さんに日程確認」
    → タスクやな → Todoist #マルタ村 に移動
    → tasks/current.md にも追加

  「〇〇の大会のルールが変わったらしい」
    → 知識やな → sp-brain/knowledge/molkky/ に保存
    → Todoist からは削除（タスクじゃないから）

  「ししょーが新しいアプリ作りたいって言ってた」
    → 人物メモやな → sp-brain/memory/people/shisho.md に追記
    → 次のチャプチェ会のアジェンダ候補として記録
    → Todoist からは削除

判定基準:
  - 動詞がある（「やる」「確認する」「連絡する」等）→ タスク
  - 情報・事実（「〇〇らしい」「〇〇が変わった」）→ 知識
  - 人に関すること → 人物メモ
  - 判断・方針（「〇〇した方がいい」「〇〇はやめとく」）→ 判断ログ
```

## Todoist プロジェクト構成

| プロジェクト | 用途 | 優先度 |
|------------|------|--------|
| Inbox | 全部ここに入れる。AI秘書が振り分ける | — |
| MOLKKY HUB | HUB記事・SEO・アフィリエイト | 最高 |
| マルタ村 | チャプチェ会・アプリ・中さん連携 | 高 |
| 収益化 | BOOTH/note/Crowdworks | 中 |
| 開発 | GitHub Pages/GAS/Vercel | 中 |
| SPクローン | AI秘書システム自体の改善 | 低 |

## タスクのライフサイクル

```
Inbox → [AI秘書が振り分け] → プロジェクト → [実行] → 完了

1. SPがTodoistにメモ（3秒）
2. セッション開始 → AI秘書が Inbox を取得
3. 分類して適切なプロジェクトに移動
4. tasks/current.md に反映（今日/今週/いつか）
5. 実行（AI秘書 or SP）
6. 完了 → Todoist も tasks/current.md も更新
```

---

## 同期ルール（Todoist ↔ tasks/current.md）

- **Todoist が正**: SPが直接 Todoist を操作することがある
- **tasks/current.md は副本**: セッション開始時に Todoist から同期
- **完了の反映**: Claude Code でタスク完了 → Todoist も完了に
- **衝突時**: Todoist 側を優先

---

## リマインド

### AI秘書がやるリマインド
- Todoist p1 タスクが期限切れ → ブリーフィングでハイライト
- Inbox に3日以上放置されたメモ → 強制的に分類
- 「今週やること」が金曜に残っている → 週次レポートでハイライト

### Todoist のリマインド（Pro版のみ）
- 期限前にプッシュ通知
- 無料版でも期限当日は通知される
