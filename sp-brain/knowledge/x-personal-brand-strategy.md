## 1. ペルソナ設計

### SP Projects とは何者か

**一言**: AIと自動化で複数事業を1人で回してる関西の事業者

**ストーリー**:
「1日30分の作業で5つのWebメディアを運営してる。AIに社長をやらせてて、日報も記事も大会情報も全部自動。モルックっていうフィンランドのスポーツにハマって、気づいたら日本最大のモルックメディアを作ってた。全部Claude Codeとスプレッドシートとサーバーレスで動いてる。完璧は目指さん。70点で出して改善する。」

### キャラクター特性

| 項目 | 内容 |
|------|------|
| **表示名** | SP Projects｜AI×モルック×メディア |
| **ハンドル** | @sp_projects（バックアップ: @sp_projects_, @spprojects） |
| **人物像** | 関西在住の事業者。顔出しなし。複数メディア運営 |
| **口調** | 関西弁ベース。「〜やねん」「ええと思う」「動けばOK」 |
| **思想** | 70点主義 / 自動化できんなら後回し / 手間をかけない |
| **強み** | AI自動化の実践者（理論じゃなく実際にやってる） |
| **弱み（見せる）** | WAFに弾かれる、自動化したら逆に手間増えた、等 |
| **言語** | 日本語メイン（技術用語は英語OK） |

### 「なにそれ」フック（初見で気になるポイント）

1. **「AIに社長やらせてます」** — AI-SP体制。組織図まである
2. **「1人なのに日報がある」** — しかも自動生成。誰に報告してんねん
3. **「モルックって何？で日本最大のメディア作った」** — ニッチ×AI
4. **「5メディア同時運営で1日30分」** — 嘘やろ？→証拠見せる
5. **「日報をAIが書いて、AIが読んで、AIが次のタスク決める」** — 無限ループ感
6. **「40個のWebアプリを全部バニラJSで作った」** — React使わん派

### ターゲット層

**コア**: 副業・個人開発をやってる（orやりたい）人
**拡張**: AI活用に興味あるビジネスパーソン、indie hackers
**ニッチ**: モルック関係者（小さいけど熱量高い）

**刺さるポイント別**:
- 「自動化したい人」→ 仕組みの具体的な話
- 「副業で稼ぎたい人」→ 収益レンジの公開
- 「AI使いこなしたい人」→ Claude Code/GASの実践Tips
- 「モルック好き」→ モルハブの裏側

## 2. 開示設計：3層モデル

### Layer 1: 全開（これがフック）
- 自動化の仕組みの詳細（Claude Code, GAS, Firebase, Vercel）
- 「5メディア同時運営」「1日30分」の事実と証拠
- 失敗談（API設計ミス、WAFに弾かれた、自動化の罠）
- 70点主義の哲学、判断基準
- AI活用のリアルな過程（スクショ付き）
- AI-SP体制の全容（組織図、エスカレーションルール）
- 日報自動生成の仕組み
- モルハブの運営者であること
- 使ってるツール・技術スタックの全容
- Claude Code Web / CLI / Coworkの使い分け

### Layer 2: ざっくり（信頼構築）
- PV/収益はレンジで（「月5桁」「月4桁PV」→成長率で語る）
- 事業のジャンル（モルック、AI、生活系メディア）
- チーム体制（「コアメンバー数人で回してる」）
- 月次・週次の振り返り数字

### Layer 3: 非公開（絶対出さない）
- 本名、顔、本業
- モルハブ以外のメディア名・URL（後出しで順次公開はあり）
- 具体的な収益内訳（メディア別の数字）
- パートナー・クライアントの個人情報
- モルックドームの詳細（タイミング見て出す可能性あり）

## 4. コンテンツ戦略

### コンテンツの7本柱

#### 柱1: 「今日自動化したこと」（メイン・週3-4回）
日報ログから自動抽出 → ツイート化

**具体ネタ例**:
- 「GASで大会情報を自動スクレイピング→WP投稿まで組んだ。所要時間2時間、これで毎週の手動作業30分がゼロに」
- 「Claude Codeで記事5本を30分で生成→投稿した。プロンプトはこう書いてる（スクショ）」
- 「Firebaseのキューに議事録を突っ込んだら、要約→Notion投稿まで自動で走るようにした」
- 「BuildHubの日次記事配信、Reddit/HN/Zenn/Qiitaから自動収集→Gemini要約→朝7時に自動公開。完全放置」

**型テンプレ**:
```
【今日の自動化】
やったこと: ◯◯
Before: △△（手動で◯分かかってた）
After: ◯◯（自動で△秒）
使ったもの: Claude Code / GAS / Firebase
ハマったとこ: ◯◯
```

#### 柱2: 「AI秘書（もう1人のSP）」シリーズ（週1-2回）
AI-SP体制の面白さ・異常さを出す

**具体ネタ例**:
- 「うちの会社、社長がAIです。組織図も作ってあって、5人の部長AIがいます。従業員は僕1人」
- 「AIに日報を書かせてるんやけど、その日報をAIが読んで明日のタスク決めてる。俺いる？」
- 「AI秘書のエスカレーションルール：Lv.0は勝手にやれ、Lv.1は報告だけしろ、Lv.2は24時間以内に聞け、Lv.3は今すぐ聞け。5000円以上の支出はLv.3」
- 「セッション開始すると、AIが『おはようございます。昨日の未完了タスクはこれです。今日何します？』って聞いてくる。上司やん」
- 「AI秘書の記憶システム：短期記憶（セッションログ）→長期記憶（週次蒸留）→忘却（月次アーカイブ）。人間の脳と同じ構造にしてる」

**型テンプレ**:
```
【AI秘書日記】
今日AIがやったこと: ◯◯
俺がやったこと: △△
感想: ◯◯
```

#### 柱3: 「Claude Code実践ガイド」シリーズ（週1-2回）
Claude解説＋図解。有用性で拡散を狙う

**具体ネタ例**:
- 「Claude Code、3つの使い方があるの知ってた？Web版 / CLI版 / Cowork。それぞれ全然違う。図解する👇」
- 「Claude CodeのCLI版、ターミナルからgit操作もファイル編集もデプロイも全部やってくれる。VSCode要らん」
- 「Claude Codeに『CLAUDE.md』っていうルールファイルを渡すと、毎回同じ判断基準で動いてくれる。うちの200行ある」
- 「Claude Code × GASの組み合わせが最強すぎる件。記事生成→画像取得→WP投稿→SNS共有まで全部自動」
- 「Claude Codeのサブエージェント機能、Haiku（安い方）に単純作業やらせてOpus（賢い方）は判断だけ。コスト1/10」

**型テンプレ（図解スレッド用）**:
```
Claude Codeの◯◯、こう使うと△△できる。

🔹 ステップ1: ◯◯
🔹 ステップ2: △△
🔹 ステップ3: ◯◯

実際のコマンド/画面はこれ👇（スクショ）
```

#### 柱4: 「モルック × テック」（週1回）
モルックを入口に、ニッチ×テックの面白さを出す

**具体ネタ例**:
- 「モルックって知ってる？フィンランド発のスポーツで、日本でいま急成長中。その日本最大のWebメディアを1人で運営してる」
- 「MOLKKY HUB、月間3000セッション超えた。AI記事生成＋7人のライターチーム＋大会情報自動投稿で回してる」
- 「マイナースポーツのメディアってブルーオーシャンすぎる。競合ほぼゼロやから、SEO楽勝」
- 「モルックの大会情報、GASで自動収集→自動投稿してる。手動だと毎週30分かかってた作業がゼロ」

#### 柱5: 「数字で振り返り」（月1-2回）
収益・PV・成長率をレンジで公開

**具体ネタ例**:
- 「【月次報告】2月のPV: ◯千 → 3月: ◯千（+◯%）。やったこと: ◯◯」
- 「5メディア合計で月◯桁到達。内訳は言えんけど、1番伸びてるのはモルハブ」
- 「自動化に投資した時間: ◯時間。それで削減できた月間作業時間: ◯時間。ROI: ◯倍」
- 「今月のClaude API代: ◯円。生成した記事数: ◯本。1記事あたり◯円」

**型テンプレ**:
```
【◯月の振り返り】
📊 PV: ◯◯（前月比+◯%）
💰 収益レンジ: 月◯桁
📝 公開記事数: ◯本
🤖 自動化した作業: ◯個
💡 今月の学び: ◯◯
```

#### 柱6: 「70点哲学 / 思考メモ」（週1-2回）
共感・思想系。引用RT・いいね狙い

**具体ネタ例**:
- 「完璧を目指すな。動くものを出せ。3時間悩むなら30分で作って出して反応見ろ」
- 「自動化できんなら後回し。手動でやるのは最後の手段」
- 「ReactもVueも使ってない。バニラJSで40アプリ作った。フレームワーク選定に3日使うなら、その3日で完成する」
- 「投資基準: 月5000円以下→即GO。超えたら一回考える。考えるのは超えてからでいい」
- 「マイナージャンルを攻めろ。競合がいないところで1位を取るのが最短ルート」
- 「AIに仕事を奪われる？いや、AIに仕事をやらせろ。奪われるんじゃなくて渡すんや」

#### 柱7: 「失敗談・やらかし」（週1回）
人間味＋高エンゲージメント

**具体ネタ例**:
- 「WAFに弾かれて3時間溶けた話。WordPress REST APIに連続POSTしたら、ConoHaのWAFが『攻撃か？』って止めてきた。10秒間隔にしたら解決」
- 「自動化したら逆に手間が増えたパターン。GASの定期実行がコケたときの調査に毎回30分取られる。手動のほうが早い説ある」
- 「AIに記事書かせたら『いかがでしたか？』って入ってた。NGワードリスト作った」
- 「Firebaseの無料枠を超えそうになって夜中に起きた話」
- 「Claude Codeにmainブランチに直接pushさせてしまって、PRフロー全部すっ飛ばした話」

## 5. 日報→ツイート変換パイプライン

### 仕組み
毎日の活動ログ（Firebase `/daily-activities/YYYY-MM-DD`）と
GAS日報（コミット＋活動ログのGemini要約）を活用

### フロー
```
日報データ（Firebase）
  ↓
ツイートネタ候補を自動生成（Gemini/Claude）
  ↓
5本のツイート案をリスト化
  ↓
SPが選ぶ or そのまま投稿
```

### 変換ルール
- 技術的な作業 → 柱1「今日自動化したこと」形式
- AI関連の作業 → 柱2「AI秘書」or 柱3「Claude実践」形式
- 数字が出てる → 柱5「数字で振り返り」形式
- トラブル/バグ → 柱7「失敗談」形式
- それ以外 → 柱6「70点哲学」のネタとしてストック

## 6. 投稿オペレーション

### 投稿ペース
- **最低**: 3ツイート/日（朝1、昼1、夜1）
- **通常**: 5ツイート/日（オリジナル3 + リプ/引用2）
- **理想**: 8ツイート/日（オリジナル5 + リプ/引用3）
- **週1**: スレッド（図解 or 深掘り）

### 時間配分（1日15-30分）
- **朝（5分）**: 日報ベースのツイート1本 + TL確認
- **昼（5分）**: Tips or 気づきを1本
- **夜（10-20分）**: 今日の成果 + リプ/引用RT + 明日のネタ確認

### 曜日別テーマ（目安）
| 曜日 | メインテーマ | 柱 |
|------|-------------|-----|
| 月 | 週の目標宣言 + 先週の振り返り | 5 |
| 火 | 自動化・開発ネタ | 1 |
| 水 | Claude Code解説 | 3 |
| 木 | AI秘書ネタ or モルック | 2 or 4 |
| 金 | 失敗談 or 70点哲学 | 7 or 6 |
| 土 | フリー（反応良かったテーマの深掘り） | - |
| 日 | スレッド（週1の図解 or まとめ） | 3 |

## 7. 成長戦略

### Phase 1: 0→500フォロワー（最初の3ヶ月）
**目標**: 「何者か」が伝わる状態を作る

- 毎日投稿を習慣化（最低3ツイート/日）
- indie hacker / AI活用 / 個人開発界隈にリプ（1日5件以上）
- ハッシュタグ: `#buildinpublic` `#個人開発` `#ClaudeCode` `#AI自動化`
- 週1でスレッド（図解がバズりやすい）
- モルック界隈への露出（MOLKKY HUBとの相互送客）
- 固定ツイートで「何者か」を明確に

**Phase 1で出すスレッド案**:
1. 「AIで5メディア同時運営してる仕組みを全公開する」
2. 「Claude Code Web / CLI / Coworkの違いを図解する」
3. 「1人会社にAI社長を置いた結果」
4. 「月3000セッションのモルックメディアをAIで作った話」
5. 「バニラJSで40アプリ作った理由（React使わない）」

### Phase 2: 500→3,000（3-6ヶ月）
**目標**: コンテンツの型が固まり、安定的に伸びる

- 反応が良い柱に比率を寄せる
- 他のクリエイターとのコラボ/メンション
- 月次レポート定例化（数字で語る）
- note記事への導線を作る（無料で出して認知拡大）
- 「SP Projects = AI自動化の実践者」のポジション確立

### Phase 3: 3,000→10,000（6-12ヶ月）
**目標**: マネタイズ開始 + ポジション確立

- **note有料マガジン**: 自動化テンプレ、具体的な収益数字、裏側の詳細
- **LINE公式 or メルマガ**: クローズドな情報配信
- X Premium加入（収益化・インプレッション優遇）
- 「AI自動化の人」としてメディア露出・登壇
- モルック×テックのユニークポジション確立

### マネタイズ設計
| レイヤー | 内容 | 価格帯 |
|----------|------|--------|
| **無料（X）** | プロセス公開、Tips、失敗談 | ¥0 |
| **無料（note）** | 長文解説、図解記事 | ¥0 |
| **有料（note）** | 具体的な数字、テンプレ、自動化の裏側詳細 | ¥500-1,500/本 |
| **有料（LINE/メルマガ）** | 週次の詳細レポート、質問対応 | ¥980-2,980/月 |

### SPのユニークポジション
上記の誰とも被らない独自性:
- **「AIに社長やらせてる」** → 誰もやってない

- **「匿名×事業者」** → 関西弁の人間味で匿名感を薄める
- **「全自動×手動ゼロ宣言」** → 証拠付きで見せるから信頼される

## 9. やらないこと

- 顔出し、本名公開

- AI臭い定型文（「いかがでしたか？」は即死）
- フォロワー購入、相互フォロー乞食
- 宣伝ばかりのタイムライン
- 根拠のない数字・盛った実績
- 他メディアアカウントとの直接リンク（当面）
