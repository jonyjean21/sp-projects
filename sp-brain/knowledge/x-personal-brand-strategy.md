## 1. ペルソナ設計

### SP Projects とは何者か

**一言**: 本業の傍ら、AIを右腕にして複数事業を本気で回してる関西の副業マン

**ストーリー**:
「副業で5つのWebメディアを運営してる。Claude Codeを右腕にして、記事生成も大会情報の更新も日報も全部自動化した。モルックっていうフィンランドのスポーツにハマって、気づいたら日本最大のモルックメディアを作ってた。ここまでやってきた過程が全部ネタになる。完璧は目指さん。70点で出して改善する。本気で月20万を目指して動いてる最中。」

### キャラクター特性

| 項目 | 内容 |
|------|------|
| **表示名** | SP Projects｜副業×AI×メディア |
| **ハンドル** | @sp_projects（バックアップ: @sp_projects_, @spprojects） |
| **人物像** | 関西在住の副業マン。顔出しなし。AIを右腕に複数メディア運営 |
| **口調** | 関西弁ベース。「〜やねん」「ええと思う」「動けばOK」 |
| **思想** | 70点主義 / 自動化できんなら後回し / 手間をかけない |
| **強み** | AI自動化の実践者（理論じゃなく実際にやってる） |
| **弱み（見せる）** | WAFに弾かれる、自動化したら逆に手間増えた、等 |
| **言語** | 日本語メイン（技術用語は英語OK） |

### 「なにそれ」フック（初見で気になるポイント）

1. **「副業で月20万目指して本気で動いてる」** — リアルタイムの挑戦。結果がどうなるか分からん臨場感
2. **「1人で5メディア同時運営」** — しかも本業ありで。どうやってんねん
3. **「モルックって何？で日本最大のメディア作った」** — ニッチ×テック
4. **「AIを右腕にしたら作業時間が1日30分になった」** — 嘘やろ？→証拠見せる
5. **「1人なのに日報がある」** — しかも自動生成。AIが書いてAIが読む
6. **「40個のWebアプリを全部バニラJSで作った」** — React使わん派

### ターゲット層

**コア**: 副業で本気で稼ぎたい人 / 個人開発やってる（orやりたい）人
**拡張**: AI活用に興味あるビジネスパーソン、indie hackers
**ニッチ**: モルック関係者（小さいけど熱量高い）

**刺さるポイント別**:
- 「副業で稼ぎたい人」→ 月20万への挑戦過程、収益レンジの公開
- 「自動化したい人」→ 仕組みの具体的な話（Before/After付き）
- 「AI使いこなしたい人」→ Claude Code/GASの実践Tips
- 「モルック好き」→ モルハブの裏側
- 「何か始めたいけど踏み出せない人」→ 70点で出す姿勢がきっかけに

## 2. 開示設計：3層モデル

### Layer 1: 全開（これがフック）
- 自動化の仕組みの詳細（Claude Code, GAS, Firebase, Vercel）
- 「5メディア同時運営」「1日30分」の事実と証拠
- 失敗談（API設計ミス、WAFに弾かれた、自動化の罠）
- 70点主義の哲学、判断基準
- AI活用のリアルな過程（スクショ付き）
- AIを右腕にした運用体制の全容（エスカレーションルール等）
- 日報自動生成の仕組み
- モルハブの運営者であること
- 使ってるツール・技術スタックの全容
- Claude Code Web / CLI / Coworkの使い分け

### Layer 2: ざっくり（信頼構築）
- PV/収益はレンジで（「月5桁」「月4桁PV」→成長率で語る）
- 事業のジャンル（モルック、AI、生活系メディア）
- チーム体制（「コアメンバー数人で回してる」）
- 月次・週次の振り返り数字

### Layer 3: 非公開（絶対出さない）
- 本名、顔、本業
- モルハブ以外のメディア名・URL（後出しで順次公開はあり）
- 具体的な収益内訳（メディア別の数字）
- パートナー・クライアントの個人情報
- モルックドームの詳細（タイミング見て出す可能性あり）

## 4. コンテンツ戦略

### コンテンツの7本柱

#### 柱1: 「今日自動化したこと」（メイン・週3-4回）
日報ログから自動抽出 → ツイート化

**具体ネタ例**:
- 「GASで大会情報を自動スクレイピング→WP投稿まで組んだ。所要時間2時間、これで毎週の手動作業30分がゼロに」
- 「Claude Codeで記事5本を30分で生成→投稿した。プロンプトはこう書いてる（スクショ）」
- 「Firebaseのキューに議事録を突っ込んだら、要約→Notion投稿まで自動で走るようにした」
- 「BuildHubの日次記事配信、Reddit/HN/Zenn/Qiitaから自動収集→Gemini要約→朝7時に自動公開。完全放置」

**型テンプレ**:
```
【今日の自動化】
やったこと: ◯◯
Before: △△（手動で◯分かかってた）
After: ◯◯（自動で△秒）
使ったもの: Claude Code / GAS / Firebase
ハマったとこ: ◯◯
```

#### 柱2: 「AIを右腕にした副業術」シリーズ（週1-2回）
Claude Codeとの二人三脚で事業を回すリアルを出す

**具体ネタ例**:
- 「Claude Codeに日報を書かせてるんやけど、それを翌日のセッションで読み込ませたら前回の続きから完璧に再開する。引き継ぎゼロ」
- 「AIにエスカレーションルール作った。Lv.0: 勝手にやれ、Lv.1: 報告だけ、Lv.2: 24時間以内に聞け、Lv.3: 今すぐ聞け。5000円以上はLv.3」
- 「AIの記憶システム作ってる。短期記憶→週次蒸留→月次アーカイブ。1ヶ月前のことも覚えてる右腕」
- 「セッション開始すると、AIが昨日の未完了タスクと今日の優先度を出してくれる。自分でタスク管理する必要がない」
- 「副業で一番しんどいのは時間がないこと。だからAIに判断以外の全部を渡した。記事書く、投稿する、データ集める、全部」

**型テンプレ**:
```
【AIとの今日の作業】
AIがやったこと: ◯◯
俺がやったこと: △△（判断だけ）
かかった時間: ◯分
感想: ◯◯
```

#### 柱3: 「Claude Code実践ガイド」シリーズ（週1-2回）
Claude解説＋図解。有用性で拡散を狙う

**具体ネタ例**:
- 「Claude Code、3つの使い方があるの知ってた？Web版 / CLI版 / Cowork。それぞれ全然違う。図解する👇」
- 「Claude CodeのCLI版、ターミナルからgit操作もファイル編集もデプロイも全部やってくれる。VSCode要らん」
- 「Claude Codeに『CLAUDE.md』っていうルールファイルを渡すと、毎回同じ判断基準で動いてくれる。うちの200行ある」
- 「Claude Code × GASの組み合わせが最強すぎる件。記事生成→画像取得→WP投稿→SNS共有まで全部自動」
- 「Claude Codeのサブエージェント機能、Haiku（安い方）に単純作業やらせてOpus（賢い方）は判断だけ。コスト1/10」

**型テンプレ（図解スレッド用）**:
```
Claude Codeの◯◯、こう使うと△△できる。

🔹 ステップ1: ◯◯
🔹 ステップ2: △△
🔹 ステップ3: ◯◯

実際のコマンド/画面はこれ👇（スクショ）
```

#### 柱4: 「モルック × テック」（週1回）
モルックを入口に、ニッチ×テックの面白さを出す

**具体ネタ例**:
- 「モルックって知ってる？フィンランド発のスポーツで、日本でいま急成長中。その日本最大のWebメディアを1人で運営してる」
- 「MOLKKY HUB、月間3000セッション超えた。AI記事生成＋7人のライターチーム＋大会情報自動投稿で回してる」
- 「マイナースポーツのメディアってブルーオーシャンすぎる。競合ほぼゼロやから、SEO楽勝」
- 「モルックの大会情報、GASで自動収集→自動投稿してる。手動だと毎週30分かかってた作業がゼロ」

#### 柱5: 「数字で振り返り」（月1-2回）
収益・PV・成長率をレンジで公開

**具体ネタ例**:

- 「今月のClaude API代: ◯円。生成した記事数: ◯本。1記事あたり◯円」

**型テンプレ**:
```
【◯月の振り返り】

📝 公開記事数: ◯本
```

#### 柱6: 「70点哲学 / 思考メモ」（週1-2回）
共感・思想系。引用RT・いいね狙い

**具体ネタ例**:
- 「完璧を目指すな。動くものを出せ。3時間悩むなら30分で作って出して反応見ろ」

## 5. 日報→ツイート変換パイプライン

### 仕組み
毎日の活動ログ（Firebase `/daily-activities/YYYY-MM-DD`）と
GAS日報（コミット＋活動ログのGemini要約）を活用

### フロー
```
日報データ（Firebase）
  ↓
ツイートネタ候補を自動生成（Gemini/Claude）
  ↓
5本のツイート案をリスト化
  ↓
SPが選ぶ or そのまま投稿
```

### 変換ルール
- 技術的な作業 → 柱1「今日自動化したこと」形式
- AI関連の作業 → 柱2「AIを右腕に」or 柱3「Claude実践」形式
- 数字が出てる → 柱5「数字で振り返り」形式
- トラブル/バグ → 柱7「失敗談」形式
- それ以外 → 柱6「70点哲学」のネタとしてストック

## 6. 投稿オペレーション

### 投稿ペース
- **最低**: 3ツイート/日（朝1、昼1、夜1）
- **通常**: 5ツイート/日（オリジナル3 + リプ/引用2）
- **理想**: 8ツイート/日（オリジナル5 + リプ/引用3）
- **週1**: スレッド（図解 or 深掘り）

### 時間配分（1日15-30分）
- **朝（5分）**: 日報ベースのツイート1本 + TL確認
- **昼（5分）**: Tips or 気づきを1本
- **夜（10-20分）**: 今日の成果 + リプ/引用RT + 明日のネタ確認

## 7. 成長戦略

### Phase 1: 0→500フォロワー（最初の3ヶ月）
**目標**: 「何者か」が伝わる状態を作る

- 毎日投稿を習慣化（最低3ツイート/日）
- indie hacker / AI活用 / 個人開発界隈にリプ（1日5件以上）
- ハッシュタグ: `#buildinpublic` `#個人開発` `#ClaudeCode` `#AI自動化`
- 週1でスレッド（図解がバズりやすい）
- モルック界隈への露出（MOLKKY HUBとの相互送客）
- 固定ツイートで「何者か」を明確に

**Phase 1で出すスレッド案**:
1. 「副業で5メディア同時運営してる仕組みを全公開する」
2. 「Claude Code Web / CLI / Coworkの違いを図解する」
3. 「副業マンがAIを右腕にしたら作業時間が1日30分になった話」
4. 「月3000セッションのモルックメディアをAIで作った話」
5. 「バニラJSで40アプリ作った理由（React使わない）」
6. 「ここまでの副業ジャーニーを全部振り返る（日報から再構成）」

### Phase 2: 500→3,000（3-6ヶ月）
**目標**: コンテンツの型が固まり、安定的に伸びる

- 反応が良い柱に比率を寄せる
- 他のクリエイターとのコラボ/メンション
- 月次レポート定例化（数字で語る）
- note記事への導線を作る（無料で出して認知拡大）
- 「SP Projects = 副業×AI自動化の実践者」のポジション確立

### Phase 3: 3,000→10,000（6-12ヶ月）
**目標**: マネタイズ開始 + ポジション確立

- **note有料マガジン**: 自動化テンプレ、具体的な収益数字、裏側の詳細
- **LINE公式 or メルマガ**: クローズドな情報配信
- X Premium加入（収益化・インプレッション優遇）
- 「副業×AI自動化の人」としてメディア露出・登壇
- モルック×テックのユニークポジション確立
- 月20万達成 or 未達のリアルな結果報告（どっちでもコンテンツになる）

### マネタイズ設計
| レイヤー | 内容 | 価格帯 |
|----------|------|--------|
| **無料（X）** | プロセス公開、Tips、失敗談 | ¥0 |
| **無料（note）** | 長文解説、図解記事 | ¥0 |
| **有料（note）** | 具体的な数字、テンプレ、自動化の裏側詳細 | ¥500-1,500/本 |
| **有料（LINE/メルマガ）** | 週次の詳細レポート、質問対応 | ¥980-2,980/月 |

### 日報からのネタ出し（これが最大の武器）
SPがここまでやってきた全ての活動が日報に蓄積されてる。これを遡ってネタ化する：

**過去の日報から引っ張れるネタ例**:
- メディア立ち上げの過程（5サイト分のストーリー）
- 自動化パイプラインを1つずつ構築した記録
- トラブルシューティング（WAF、API制限、無料枠の壁）
- 収益化の試行錯誤（アフィリエイト設定、ASP選定）
- ツール選定の判断（なぜバニラJS、なぜFirebase、なぜVercel）
- チーム運営の学び（ライター7人の管理、モルックドームのチーム）
- 収益の成長曲線（月3000円からの推移）

**ルール**: 過去の日報を週2-3本のペースでツイート化。新しいネタと混ぜる。
ストックが膨大やから、ネタ切れは絶対にない。
