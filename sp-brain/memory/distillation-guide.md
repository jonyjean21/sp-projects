# 記憶の蒸留（Distillation）ガイド

## 概要

セッションログ・decisions・learningsは日々増える。
放置すると検索品質が落ち、コンテキストウィンドウを圧迫する。
**定期的に「蒸留」して、MEMORY.mdに重要なエッセンスだけ残す。**

## 蒸留の3ステップ

### Step 1: 収集（週次 / 月曜朝）
```
対象ファイル:
- memory/sessions/YYYYMMDD-*.md（先週分）
- memory/decisions/YYYYMMDD-*.md（先週分）
- memory/learnings/YYYYMMDD-*.md（先週分）
```

### Step 2: 判定（各エントリに対して）

| 判定 | 基準 | アクション |
|------|------|-----------|
| **蒸留** | 今後のセッションで繰り返し参照する | MEMORY.md の該当セクションに追記 |
| **保持** | まだ鮮度がある（1ヶ月以内の判断など） | そのまま残す |
| **アーカイブ** | 1ヶ月以上前 & 蒸留済み | memory/archive/ に移動 |
| **削除** | 一時的なメモ、蒸留済みの生ログ | 削除（gitに履歴は残る） |

### Step 3: 書き込み

MEMORY.md に追記する際のルール:
- **1エントリ = 1-2行**。詳細は元ファイルへのリンクで。
- **日付を必ず付ける**（いつの判断/学びか分かるように）
- **カテゴリ別に整理**: 技術 / プロジェクト / 人間関係 / 失敗

## 鮮度管理ルール

| 経過期間 | ステータス | アクション |
|----------|-----------|-----------|
| 0-1週間 | 🟢 Fresh | sessions/ にそのまま |
| 1-4週間 | 🟡 Aging | 蒸留判定の対象 |
| 1ヶ月超 | 🔴 Stale | 蒸留済みならarchive、未蒸留なら強制蒸留 |

## archive/ のルール
- `memory/archive/YYYY-MM/` にまとめて格納
- 基本的に読み返さない。必要な時だけgit logで探す
- 3ヶ月以上前のarchiveは削除してもOK（git履歴に残る）
