# 記憶蒸留（Distillation）システムの導入

- **日付**: 2026-03-10
- **決定**: セッションログ + 週次蒸留 + MEMORY.md の3層記憶システムを導入

## 背景
- sp-brain/memory/ にdecisions/learningsが溜まっていくが、整理・圧縮の仕組みがなかった
- 業界調査の結果、「生ログ → 蒸留 → 長期記憶」の3層パターンが標準的と判明
- OpenClaw、DiffMem、memsearch等の主要プロジェクトが同様のアプローチ

## 設計
1. **sessions/（生ログ）**: セッションごとの全記録。テンプレートあり
2. **decisions/ & learnings/（中期記憶）**: 重要な判断と学び。個別ファイル
3. **MEMORY.md（蒸留済み長期記憶）**: 本当に重要なエッセンスだけ。1-2行/エントリ

## 蒸留タイミング
- 週次（月曜朝のルーティン）で手動トリガー
- Phase 3（自律稼働）で自動化予定

## 他の選択肢
- ベクトルDB（Mem0等）→ オーバーエンジニアリング。ファイルベースで十分
- 蒸留なしで全保存 → コンテキストウィンドウ圧迫。検索品質低下
- 自動蒸留（今すぐ）→ Phase 3の範囲。現時点では手動で十分
