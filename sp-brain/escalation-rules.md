# エスカレーションルール

## 原則
**AI秘書は自分で判断できることは全部自分でやる。SPに相談するのは本当に必要な時だけ。**

---

## レベル定義

### Lv.0: 自動処理（SPに通知しない）
- Firebaseキューの定型処理（議事録、大会情報）
- commitしてpush
- タスクの棚卸・整理
- コンテキストファイルの更新
- 定型的なファイル編集

### Lv.1: INFO（LINEで報告のみ）
- 日次サマリー「今日これやったで」
- キュー処理完了の通知
- 週次レポート

### Lv.2: REVIEW（LINEで確認依頼）
- 記事の下書き完了「確認してや」
- 新機能の提案「これどう思う？」
- 収益レポート「数字こうなってる」
- 返信期限: 急ぎではない（24時間以内）

### Lv.3: URGENT（LINEで即通知）
- エラーが発生して自力復旧できない
- セキュリティに関わる問題
- 外部からの問い合わせ（パートナー等）
- 月5,000円を超える投資判断
- mainブランチへのマージ承認

---

## 判断フローチャート

```
新しいタスク or 判断が必要
  │
  ├─ CLAUDE.md / sp-brain/ に答えがある？
  │   ├─ YES → Lv.0: 自分でやる
  │   └─ NO ↓
  │
  ├─ 過去に似た判断をしたことがある？（memory/decisions/）
  │   ├─ YES → Lv.0: 過去の判断に従う
  │   └─ NO ↓
  │
  ├─ 失敗しても取り返しがつく？
  │   ├─ YES → Lv.1: やってから報告
  │   └─ NO ↓
  │
  ├─ お金・セキュリティ・対外関係に影響する？
  │   ├─ YES → Lv.3: URGENT で相談
  │   └─ NO → Lv.2: REVIEW で確認
  │
  └─ 判断に迷ったら → Lv.2: REVIEW で相談
```

## 具体例

| 場面 | レベル | 理由 |
|------|--------|------|
| 大会情報をWPに自動投稿 | Lv.0 | 定型処理 |
| 新しいGitHub Pagesアプリを作成 | Lv.2 | SPの確認が必要 |
| アフィリエイトリンクの更新 | Lv.0 | 定型処理 |
| 月額¥3,000のツール導入提案 | Lv.0 | ¥5,000以下なので自動承認 |
| 月額¥8,000のツール導入提案 | Lv.3 | ¥5,000超えなのでSP確認 |
| Firebase障害でキュー処理失敗 | Lv.3 | 自力復旧できない |
| 記事5本の下書き完了 | Lv.2 | SPの品質チェック |
| X投稿の自動生成 | Lv.1 | 定型だが報告はする |
| パートナーからの問い合わせ | Lv.3 | 対外関係 |
