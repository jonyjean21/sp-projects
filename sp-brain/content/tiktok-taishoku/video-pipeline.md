# TikTok動画 完全自動生成パイプライン

> 作成日: 2026-03-13（最終更新: 同日・技術リサーチ反映）
> 目的: 退職代行TikTokの動画を**月額$0で完全自動量産**する
> 方針転換: Vrewは**APIなし = 自動化不可**。Python自作パイプラインに一本化

---

## アーキテクチャ全体図

```
[1. コンテンツ戦略エンジン]
    ペルソナ×テーママトリクスからテーマを選定
    ↓
[2. 台本自動生成]
    Gemini 2.5 Flash API（無料枠: 250本/日）
    ↓ JSON出力
[3. 音声合成]
    AivisSpeech（ローカル、無料、商用OK）
    ↓ WAVファイル
[4. 動画生成]
    MoviePy + Pillow（テロップ描画）+ Pexels（背景素材）
    ↓ MP4ファイル（9:16, 1080x1920）
[5. 投稿]
    手動（TikTokアプリから1分/本）
    ※ TikTok Content Posting APIは個人bot審査が通らない
    ※ 代替: Later($25/月)で半自動スケジュール投稿
```

**月額コスト: $0**（全て無料ツール/API）
**SP作業: 投稿のみ（1本1分 × 1日1-2本 = 1日2分）**

|------|------|

| **テロップ** | 白文字 + 黒縁取り + 中央やや上配置。1画面1-2行 |

### 台本の構造（全動画共通フォーマット）


**フックのパターン集（高視聴完了率）:**
- 疑問提起型: 「退職代行の料金、知ってる？」
- 否定型: 「入社3ヶ月で辞めたいは甘えじゃない」
- 体験型: 「退職代行、マジで使ってよかった」

- 共感型: 「辞めますが言えないのはあなたが弱いんじゃない」

## 2. 台本自動生成（Gemini 2.5 Flash）

### API仕様

| 項目 | 値 |
|------|---|
| モデル | gemini-2.5-flash |
| コスト | 無料枠: $0 |
| 制限 | 10 RPM / 250 RPD |
| 台本生成能力 | 1日250本（理論値）、実用は1日10-30本で十分 |

### 台本生成プロンプト

```
あなたはTikTok動画の台本ライターです。
退職代行サービスの情報を発信する「退職のミカタ」というアカウントの台本を書いてください。

## ルール
- 尺: 15-25秒で読み切れる量（150-250文字）
- 構成: [フック 0-3秒] [共感/情報 3-12秒] [解決策 12-20秒] [CTA 20-25秒]
- フックは感情を揺さぶる一言。疑問/否定/衝撃/共感のいずれか
- 断定表現は避ける（「必ず」「絶対」→「多くの場合」「基本的に」）
- CTAは「詳しくはプロフのリンクから」で固定
- 「PR」表示が必要な旨をメタデータに含める

## 入力
- ペルソナ: {persona}
- パターン: {pattern}
- 季節テーマ: {seasonal}（あれば）

## 出力（JSON）
{
  "title": "動画タイトル",
  "hook": "フック（1文）",
  "body": "本文（3-5文）",
  "cta": "CTA（1文）",
  "full_script": "フック+本文+CTAの全文",
  "hashtags": ["#退職代行", "#ハッシュタグ2", ...],
  "caption": "TikTok投稿時のキャプション",
  "background_keywords": ["pexels検索キーワード1", "キーワード2"],
  "bgm_mood": "エモピアノ|Lo-Fi|ポップ|トレンド",
  "estimated_duration_sec": 20,
  "persona": "{persona}",
  "pattern": "{pattern}"
}
```

## 3. 音声合成（AivisSpeech）

### なぜAivisSpeechか

| 項目 | AivisSpeech | VOICEVOX | gTTS |
|------|-------------|----------|------|
| 音声品質 | ★★★★★（感情豊か） | ★★★☆☆（機械的） | ★★☆☆☆（棒読み） |
| 日本語 | ネイティブ | ネイティブ | 可だが不自然 |
| 商用利用 | OK（クレジット不要） | キャラによる | OK |
| API | VOICEVOX互換 (localhost:50021) | localhost:50021 | Python lib |
| コスト | 無料 | 無料 | 無料 |

### セットアップ

```bash
# AivisSpeechをDL（Windows/Mac/Linux）
# https://aivis-project.com/
# 起動するとlocalhost:50021でHTTP APIが立つ

# 音声合成API（VOICEVOX互換）
curl -X POST "http://localhost:50021/audio_query" \
  -H "Content-Type: application/json" \
  -d '{"text": "退職代行、マジで使ってよかった", "speaker": 0}' \
  | curl -X POST "http://localhost:50021/synthesis" \
    -H "Content-Type: application/json" \
    -d @- \
    --output voice.wav
```

### 音声スタイルの使い分け

| ペルソナ | 音声スタイル | 話速 |
|---------|------------|------|
| 新卒逃走 | 明るめ、寄り添い | やや速い |
| パワハラ限界 | 落ち着き、真剣 | 普通 |
| 業種特化 | 情報的、信頼感 | やや速い |
| バイト | カジュアル、軽め | 速い |

## 4. 動画生成（MoviePy + Pillow）

### 処理フロー

```
台本JSON
  ↓ full_scriptを読み取り
AivisSpeech API → voice.wav（音声ファイル）
  ↓ 音声の長さを取得
Pexels API → background.mp4（背景動画DL）
  ↓ background_keywordsで検索
Pillow → テロップ画像群を生成（1画面ずつ）
  ↓ フック/本文/CTAそれぞれの画像
MoviePy → 合成
  ├── 背景動画を音声の長さに合わせてトリミング
  ├── テロップ画像をタイミングに合わせて重ねる
  ├── BGMを音量20%で重ねる
  └── 9:16, 1080x1920でエンコード
  ↓
output/tiktok/{id}_{title}.mp4
output/tiktok/{id}_{title}_meta.json
```

### テロップ描画仕様（Pillow）

```python
# テロップ画像の仕様
WIDTH = 1080
HEIGHT = 1920
FONT = "NotoSansJP-Bold.otf"
FONT_SIZE = 68
TEXT_COLOR = (255, 255, 255)  # 白
STROKE_COLOR = (0, 0, 0)  # 黒縁取り
STROKE_WIDTH = 4
TEXT_Y_POSITION = HEIGHT * 0.35  # 画面の上から35%（中央やや上）
MAX_CHARS_PER_LINE = 12  # 1行の最大文字数
MAX_LINES = 3  # 最大行数
```

### BGM自動選択

```json
{
  "エモピアノ": "assets/bgm/emo_piano_01.mp3",
  "Lo-Fi": "assets/bgm/lofi_chill_01.mp3",
  "ポップ": "assets/bgm/pop_upbeat_01.mp3",
  "トレンド": "assets/bgm/trend_01.mp3"
}
```

BGMはフリー素材を事前DLしてassets/bgm/に配置。台本JSONのbgm_moodで自動選択。

### Vrewを使わなくなった理由

1. **APIが存在しない** → パイプラインに組み込めない
2. **無料枠の制限** → 月産30本以上は有料（¥1,290〜/月）
3. **バッチ処理不可** → 1本ずつGUIで手動操作が必要
4. **Python自作の方が自由度高い** → テロップスタイル、音声、BGMを完全制御可能

Vrewは手動で1-2本試す初期テスト用途には使えるが、量産パイプラインには不向き。
